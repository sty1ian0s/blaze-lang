## 1. `List<T>` ŌĆō DoublyŌĆæLinked List

### 1.1 Struct

```
pub struct List<T> {
    // private fields
}
```

A linear, doublyŌĆælinked list.  Each node is allocated in the current region.

### 1.2 Public Methods

```
impl<T> List<T> {
    pub fn new() -> Self;
    pub fn push_front(&mut self, value: T);
    pub fn push_back(&mut self, value: T);
    pub fn pop_front(&mut self) -> Option<T>;
    pub fn pop_back(&mut self) -> Option<T>;
    pub fn front(&self) -> Option<&T>;
    pub fn back(&self) -> Option<&T>;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
}
```

- `new()` creates an empty list.
- `push_front`/`push_back` move the value into a new node at the front/back.
- `pop_front`/`pop_back` remove and return the value (if any), disposing the node but not the value (the caller takes ownership).
- `front`/`back` return a reference to the head/tail value.
- `len`/`is_empty` query the number of elements.

### 1.3 Trait Implementations

```
impl<T> Dispose for List<T> {
    fn dispose(&mut self);   // iterates through nodes, disposes each element and deallocates nodes
}
impl<T> IntoIterator for List<T> {
    type Item = T;
    type IntoIter = ListIntoIter<T>;
    fn into_iter(self) -> ListIntoIter<T>;
}
```

`ListIntoIter<T>` is a linear iterator that yields each element in order, consuming the list.

## 2. `Map<K,V>` ŌĆō BŌĆæTree Map

### 2.1 Struct

```
pub struct Map<K, V> {
    // private BŌĆætree fields
}
```

An ordered map implemented as a BŌĆætree (for cacheŌĆæfriendly access).  Keys must be `Ord`.

### 2.2 Public Methods

```
impl<K: Ord, V> Map<K, V> {
    pub fn new() -> Self;
    pub fn insert(&mut self, key: K, value: V) -> Option<V>;
    pub fn get(&self, key: &K) -> Option<&V>;
    pub fn get_mut(&mut self, key: &K) -> Option<&mut V>;
    pub fn remove(&mut self, key: &K) -> Option<V>;
    pub fn contains_key(&self, key: &K) -> bool;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn iter(&self) -> MapIter<K, V>;
}
```

- `insert(key, value)`: inserts the keyŌĆævalue pair.  If the key already existed, returns the old value (`Some(old_value)`).
- `get`/`get_mut`: returns a reference to the value associated with the key.
- `remove`: removes and returns the value for the key (if present).
- `contains_key`, `len`, `is_empty`: standard queries.
- `iter()`: returns an iterator over keyŌĆævalue references (ordered by key).

### 2.3 Trait Implementations

```
impl<K: Ord, V> Dispose for Map<K, V> {
    fn dispose(&mut self);   // disposes keys, values, and internal tree nodes
}
impl<K: Ord, V> IntoIterator for Map<K, V> {
    type Item = (K, V);
    type IntoIter = MapIntoIter<K, V>;
    fn into_iter(self) -> MapIntoIter<K, V>;
}
```

`MapIter` yields `(&K, &V)` pairs.  `MapIntoIter` yields `(K, V)` pairs, consuming the map.

## 3. `SlotMap<K, T>` ŌĆō Generational Arena

### 3.1 Struct

```
pub struct SlotMap<K, T> {
    // private fields: storage, free list, generation counters
}
```

A generational arena that associates keys (opaque, copyable) with values.  Keys remain valid even after value removal, and reuse is prevented by generation indices.

### 3.2 `Key` Trait

```
pub trait Key: Copy + Eq {}
```

- Keys for a `SlotMap` must implement `Copy` and `Eq`.  They are generated by the map and are lightweight (typically two integers: index and generation).

The concrete key type `K` is defined by the mapŌĆÖs instantiation (e.g., `SlotMap<u32, T>` or a custom key).  However, to make generation automatic, we will expose a macro or a concrete `Key` struct later; for now, the user must provide a type that implements `Key`.

### 3.3 Public Methods

```
impl<K: Key, T> SlotMap<K, T> {
    pub fn new() -> Self;
    pub fn insert(&mut self, value: T) -> K;
    pub fn get(&self, key: K) -> Option<&T>;
    pub fn get_mut(&mut self, key: K) -> Option<&mut T>;
    pub fn remove(&mut self, key: K) -> Option<T>;
    pub fn contains(&self, key: K) -> bool;
    pub fn len(&self) -> usize;
}
```

- `insert(value)`: allocates a new entry, assigns a fresh key (with generation), and returns the key.
- `get`/`get_mut`: return a reference to the value if the key is valid (generation match and index occupied).
- `remove(key)`: removes the value and invalidates the key (generation bump), returns the value.
- `contains(key)`: checks validity without accessing value.
- `len`: number of currently live entries.

### 3.4 Trait Implementations

```
impl<K: Key, T> Dispose for SlotMap<K, T> { ... }
```

Disposes all values and deallocates storage.

## 4. `RingBuf<T>` ŌĆō Ring Buffer (Circular Queue)

### 4.1 Struct

```
pub struct RingBuf<T> {
    // private fields: buffer, head, tail, capacity
}
```

A fixedŌĆæcapacity, linear ring buffer.  Elements are stored in a contiguous array; insertion at back and removal from front are O(1).  When full, pushing returns `Err(T)`.

### 4.2 Public Methods

```
impl<T> RingBuf<T> {
    pub fn new(capacity: usize) -> Self;
    pub fn push_back(&mut self, value: T) -> Result<(), T>;
    pub fn pop_front(&mut self) -> Option<T>;
    pub fn front(&self) -> Option<&T>;
    pub fn back(&self) -> Option<&T>;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn capacity(&self) -> usize;
}
```

- `new(capacity)`: allocates buffer with the given capacity.
- `push_back(value)`: if the buffer is not full, moves the value into the back and returns `Ok(())`.  If full, returns `Err(value)`.
- `pop_front()`: removes and returns the front element.
- `front`/`back`: return references.
- `len`, `is_empty`, `capacity`: queries.

### 4.3 Trait Implementations

```
impl<T> Dispose for RingBuf<T> { ... }
```

Disposes all remaining elements and deallocates the buffer.
