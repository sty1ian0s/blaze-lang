Blaze Phase 3b – Systems Library: Futures

Previous: File System | Next: Platform OS

Goal

Core asynchronous types: Future, Poll, Context, Wake, and Pin.

1. std::pin – Pinning

1.1 Pin<P>

pub struct Pin<P> { pointer: P }

Wrapper that prevents moving the pointee.

1.2 Constructor

pub unsafe fn new_unchecked(pointer: P) -> Pin<P>;

1.3 Deref and DerefMut

impl<P: Deref> Deref for Pin<P> { type Target = P::Target; ... }
impl<P: DerefMut> DerefMut for Pin<P> { ... }

DerefMut does not allow moving the pinned value.

1.4 Unpin

pub auto trait Unpin {}

Auto trait; types opt out with !Unpin.

2. std::future

2.1 Future Trait

pub trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context) -> Poll<Self::Output>;
}

2.2 Poll<T>

pub enum Poll<T> { Ready(T), Pending }

2.3 Context

pub struct Context<'a> { waker: &'a dyn Wake }

2.4 Wake Trait

pub trait Wake { fn wake(self: &Self); }

3. Unsafe Future Marker

pub unsafe trait UnsafeFuture: Future {}

4. Implementation Notes

  • Compiler transforms async fn to Future state machine.
  • Context and Wake used by executor.
  • Pin is a zero‑cost wrapper.

5. Testing

  • Pin: deref works; move out disallowed.
  • Manual Future impl: poll returns Ready.
  • Async function: .poll() works.
