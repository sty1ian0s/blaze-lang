Blaze Phase 4 – Ecosystem: Postgres

Previous: Physics3D | Next: RabbitMQ

Goal

PostgreSQL driver implementing blaze‑sql traits. Native wire protocol (v3) over TCP/TLS. Async, linear types.

1. Database Type

pub struct Postgres;
impl Database for Postgres { type Connection = PostgresConnection; fn connect(uri: &str) -> Result<Self::Connection, Error>; }

2. Connection

pub struct PostgresConnection { stream: Box<dyn AsyncReadWrite>, parameters: Map<Text, Text>, state: ConnectionState }
impl Connection for PostgresConnection {
    async fn execute(&self, query: &Query) -> Result<ExecuteResult, Error>;
    async fn query(&self, query: &Query) -> Result<Rows, Error>;
    async fn transaction(&self) -> Result<Transaction<'_, Self>, Error>;
    fn is_autocommit(&self) -> bool;
    fn set_autocommit(&mut self, value: bool) -> Result<(), Error>;
    fn close(self) -> Result<(), Error>;
}

3. Transaction

pub struct PostgresTransaction<'a> { conn: &'a mut PostgresConnection, depth: u32, committed: bool }
impl<'a> PostgresTransaction<'a> {
    pub async fn commit(self) -> Result<(), Error>;
    pub async fn rollback(self) -> Result<(), Error>;
}
impl<'a> Dispose for PostgresTransaction<'a> { … }

4. Wire Protocol

Implements Startup, Parse, Bind, Execute, Sync, etc. Binary format for all types.

5. Type Mapping

Maps PostgreSQL OIDs to Blaze types: int4 → i32, text → Text, jsonb → JsonValue, etc.

6. Error Handling

pub enum PgError {
    Io(std::io::Error),
    WireFormat(Text),
    AuthFailed(Text),
    QueryError(PgErrorResponse),
    ConnectionClosed,
    Timeout,
}
pub struct PgErrorResponse { severity: Text, code: Text, message: Text, detail: Option<Text>, hint: Option<Text>, position: Option<u32> }
