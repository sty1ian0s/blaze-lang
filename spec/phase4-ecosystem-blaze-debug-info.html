Blaze Phase 4 – Ecosystem: Debug Info

Previous: Crypto | Next: Doc Gen

Goal

Provide pure library for reading/writing Blaze Debug Format (BDF), a superset of DWARF. Used by compiler, debugger, backtrace.

1. Core Types

pub struct DebugInfo {
    compile_units: Vec<CompileUnit>,
    line_program: LineProgram,
    address_ranges: Vec<AddressRange>,
    strings: StringTable,
}
pub struct CompileUnit { name: Text, language: Language, functions: Vec<FunctionDI>, variables: Vec<VariableDI>, types: Vec<TypeDI> }
pub struct LineProgram { entries: Vec<LineEntry> }
pub struct LineEntry { address: u64, file: u32, line: u32, column: u32 }

2. Parsing

pub fn read_from_bytes(data: &[u8]) -> Result<DebugInfo, DebugInfoError>;
pub fn read_from_file(path: &str) -> Result<DebugInfo, DebugInfoError>;

3. Writing

pub fn write_to_bytes(info: &DebugInfo) -> Result<Vec<u8>, DebugInfoError>;
pub fn write_to_file(info: &DebugInfo, path: &str) -> Result<(), DebugInfoError>;

4. Lookup Functions

impl DebugInfo {
    pub fn find_function(&self, address: u64) -> Option<&FunctionDI>;
    pub fn find_location(&self, address: u64) -> Option<(u32, u32, u32)>;
    pub fn find_type(&self, name: &str) -> Option<&TypeDI>;
}

5. Integration

Used by blaze‑backtrace for symbolication and blaze‑debug for source‑level debugging.

6. Error Handling

pub enum DebugInfoError {
    Io(std::io::Error),
    InvalidFormat,
    UnsupportedVersion,
    CorruptedData,
}

7. Testing

  • Round‑trip: write/read.
  • Lookup: known addresses.
  • DWARF compatibility: parse standard DWARF5.
