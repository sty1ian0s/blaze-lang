Blaze Phase 3b – Systems Library: Time

Previous: Synchronization | Next: Extended – Dynamic

Goal

Provide Duration and Instant for measuring time intervals and monotonic clocks.

1. Duration

pub struct Duration { secs: u64, nanos: u32 }

Represents span; nanos < 1_000_000_000.

Constructors

pub fn new(secs: u64, nanos: u32) -> Duration;
pub fn from_secs(secs: u64) -> Duration;
pub fn from_millis(millis: u64) -> Duration;
pub fn from_micros(micros: u64) -> Duration;
pub fn from_nanos(nanos: u64) -> Duration;

Accessors

pub fn as_secs(&self) -> u64;
pub fn subsec_millis(&self) -> u32;
pub fn subsec_micros(&self) -> u32;
pub fn subsec_nanos(&self) -> u32;

Arithmetic

Implements Add, Sub, Mul, Div, and assignment variants. Overflow panics in debug, wraps in release.

Comparisons: PartialEq, Eq, PartialOrd, Ord.

@copy, Default (zero).

2. Instant

pub struct Instant { t: u64 }

Monotonic timestamp (nanoseconds from arbitrary epoch).

Constructor

pub fn now() -> Instant;

Operations

impl Sub<Instant> for Instant { type Output = Duration; ... }
impl Add<Duration> for Instant { type Output = Instant; ... }

PartialEq, Eq, PartialOrd, Ord.

elapsed

pub fn elapsed(&self) -> Duration;

@copy.

3. Implementation Notes

  • Unix: clock_gettime(CLOCK_MONOTONIC).
  • Windows: QueryPerformanceCounter.
  • Instant stores nanoseconds; conversion from raw ticks.

4. Testing

  • Duration: arithmetic, sub‑second components, comparisons.
  • Instant: now(), sleep, elapsed within bounds.
  • Monotonicity: consecutive calls non‑decreasing.
  • Overflow: panics in debug, wraps in release.
