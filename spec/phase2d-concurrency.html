Blaze Phase 2d – Concurrency: Actors, Async, Channels, Supervision

Previous: Phase 2c | Next: Phase 2e – Contracts

Goal

Implement the actor model, asynchronous functions (async fn), channels (Sender/Receiver), and the supervision tree.

1. Actor Model

1.1 Actor Declaration

actor_decl = "actor" ident [ generic_params ] "{" { actor_item } "}"
actor_item = field_decl | fn_decl

1.2 Message Handlers (@message)

  • Method annotated @message becomes a message handler.
  • Compiler generates message struct, response handle, capability method. 
actor Counter {
    value: i32,
    @message fn inc(&mut self, n: i32) -> i32 { self.value += n; self.value }
}

1.3 Spawning Actors

spawn(actor_expression, supervisor?) → Capability<A>.

1.4 Actor Semantics

  • Cooperative: runs until await or handler completes.
  • No pre‑emption.
  • Linear: resources disposed on termination.

2. Asynchronous Functions (async fn)

2.1 Syntax and Desugaring

async fn foo() -> T returns Future<T>. await suspends until future completes.

2.2 State Machine Generation

  • Each await point becomes a state.
  • Linear variables across await must implement Dispose.
  • Generated Future provides poll.

2.3 Cancellation

Dropping a Future calls Dispose on live linear variables.

3. Channels

3.1 Channel Types

  • Sender<T> – cloneable, reference‑counted.
  • Receiver<T> – linear, implements Iterator.
  • channel<T>() -> (Sender<T>, Receiver<T>).

3.2 Semantics

  • Non‑blocking sends; receives via recv() or for loops.
  • Used for actor communication.

4. Supervisor Tree

  • Each actor has a parent supervisor.
  • If actor panics >3 times in 10 seconds, supervisor escalates to parent.
  • Root actor failure terminates process.

5. Runtime Scheduler

  • Work‑stealing thread pool (one worker per core).
  • Each worker has local deque of actor mailboxes.
  • Mailbox is lock‑free SPSC (upgraded to MPSC on clone).
  • Cooperative scheduling: run until await or handler complete.

6. Code Generation

  • State machine structures for async functions.
  • Mailboxes as ring buffers.
  • Runtime library provides thread pool, work‑stealing, supervision.

7. Testing

  • Actor messaging: spawn, send message, verify response.
  • Async/await: state machine progress, cancellation cleanup.
  • Channels: send/receive, bounded overflow, iterator.
  • Supervision: trigger panics, verify escalation and process termination.
  • Determinism: actor execution order does not affect final state.
  • Complexity: scheduler overhead O(actors). Work‑stealing scalability.
