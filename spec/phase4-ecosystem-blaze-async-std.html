Blaze Phase 4 – Ecosystem: Async Std

Previous: Arch | Next: Audio

Goal

Provide a multi‑threaded async executor, timers, I/O helpers, and synchronisation primitives for applications that don't need the full actor model.

1. Core Concepts

  • Executor – work‑stealing thread pool.
  • block_on – run future to completion.
  • spawn – spawn task onto global executor.
  • Timers: sleep, interval.
  • Async I/O: AsyncRead, AsyncWrite traits.
  • Sync primitives: async Mutex, RwLock, Semaphore, Channel.

2. Executor and Spawning

pub struct Executor { /* work‑stealing pool */ }
impl Executor {
    pub fn new() -> Result<Executor, IoError>;
    pub fn block_on<F: Future>(&self, future: F) -> F::Output;
    pub fn spawn<F: Future + Send + 'static>(&self, future: F) -> JoinHandle<F::Output>;
}
pub struct JoinHandle<T> { /* … */ }
impl<T> Future for JoinHandle<T> { type Output = T; … }

Global executor initialised lazily: spawn and block_on use it.

3. Async I/O

pub trait AsyncRead {
    async fn read(&mut self, buf: &mut [u8]) -> Result<usize, io::Error>;
    async fn read_to_end(&mut self, buf: &mut Vec<u8>) -> Result<usize, io::Error>;
}
pub trait AsyncWrite {
    async fn write(&mut self, buf: &[u8]) -> Result<usize, io::Error>;
    async fn flush(&mut self) -> Result<(), io::Error>;
}

Blanket implementations for types that implement synchronous Read/Write using blocking pool.

Wrappers: AsyncFile, AsyncTcpStream, AsyncTcpListener.

5. Synchronisation Primitives

  • Mutex<T> – async lock.
  • RwLock<T> – async read‑write.
  • Semaphore – counting semaphore.
  • Channel<T> – MPSC with async send/recv.

All linear, implement Dispose.

6. Integration with Actor Runtime

Compatible with Blaze's actor model; actors can be spawned on the same thread pool.

7. Error Handling

pub enum IoError {
    Io(std::io::Error),
    Timeout,
    Canceled,
}

8. Testing

  • Executor: spawn futures, collect results.
  • Timer: sleep wakes after duration.
  • Async I/O: write to file, read back.
  • Mutex: locked by one task at a time.
  • Channel: send/receive ordering.
