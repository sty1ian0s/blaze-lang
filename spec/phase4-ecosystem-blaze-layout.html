Blaze Phase 4 – Ecosystem: Layout

Previous: Kafka | Next: Log

Goal

Pure, data‑oriented constraint‑based layout engine for GUI. Takes widget tree and available dimensions, returns rectangles. Supports pixel, percentage, auto sizing, padding, margin, alignment, flex‑box, absolute positioning.

1. Input: LayoutNode

pub struct LayoutNode {
    pub id: u32,
    pub parent_id: Option<u32>,
    pub children_ids: Vec<u32>,
    pub constraints: LayoutConstraints,
    pub intrinsic_size: Option<(f64, f64)>,
}
pub struct LayoutConstraints {
    pub width: SizeConstraint,
    pub height: SizeConstraint,
    pub min_width: Option<f64>,
    pub max_width: Option<f64>,
    pub min_height: Option<f64>,
    pub max_height: Option<f64>,
    pub padding: EdgeInsets,
    pub margin: EdgeInsets,
    pub alignment: Alignment,
    pub flex: f64,
    pub position: Option<Position>,
}
pub enum SizeConstraint { Pixels(f64), Percent(f64), Auto }
pub struct EdgeInsets { top: f64, right: f64, bottom: f64, left: f64 }
pub struct Alignment { horizontal: HorizontalAlign, vertical: VerticalAlign }
pub enum HorizontalAlign { Left, Center, Right, Stretch }
pub enum VerticalAlign { Top, Center, Bottom, Stretch }
pub struct Position { x: f64, y: f64 }

2. Output: Rect

pub struct Rect { pub x: f64, pub y: f64, pub width: f64, pub height: f64 }

3. Solver Algorithm

Bottom‑up size computation (auto sizes, percentages) and top‑down position assignment. Single‑pass, O(N), deterministic.

4. Integration

Used by blaze‑gui.

5. Testing
