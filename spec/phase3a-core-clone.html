Blaze Phase 3a – Core Library: Clone

Previous: Builtins | Next: Comparison

Goal

Implement the std::clone module, providing the Clone trait for explicit duplication of values.

1. Clone Trait

pub trait Clone {
    fn clone(&self) -> Self;
}
  • Clone provides explicit duplication, unlike implicit @copy.
  • Types annotated @copy automatically implement Clone (trivial copy).
  • Types that implement Dispose may also implement Clone if a deep copy is possible; the new instance must be independently disposed.

2. Standard Implementations

  • All primitive types (integers, floats, bool, char, ()) → trivial copy.
  • &T → copies the reference only.
  • Text → deep copy (already implemented).
  • Option<T>, Result<T,E> → Clone if their inner types do.
  • Vec<T> → Clone if T: Clone.
  • Other collections implement Clone where appropriate.

@derive(Clone) generates an implementation that clones each field.

3. Testing

  • Primitive types clone and equal the original.
  • Text::clone() produces an independent string.
  • Custom struct with manual Clone logs calls; verify only invoked on explicit .clone().
  • Types without Clone reject .clone() calls (type checker).
