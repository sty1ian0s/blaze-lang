Blaze Phase 5 – Ecosystem: Autobalance

Previous: Asset | Next: Dataframe

Goal

Extend automatic parallelism to heterogeneous hardware (CPU, GPU, NUMA, distributed nodes). Provides a global scheduler that treats every compute resource as a ComputeDevice.

1. Core Types

pub enum ComputeDevice {
    Cpu(CpuCore),
    Gpu(GpuDevice),
    NumaNode(NumaNode),
    RemoteNode(RemoteNode),
    Custom(Box<dyn Compute>),
}

3. @balance Attribute

#[balance] fn compute_all(data: &[f64]) -> f64 { /* … */ }

Compiler generates code that packages the loop body as a Workload and submits to the global scheduler.

4. Remote Execution

Scheduler can serialise Workload (input data + closure) and send to remote runtime via actor protocol.

5. NUMA‑Aware Balancing

Queries OS for NUMA topology; memory‑intensive workloads scheduled on local NUMA node.

6. Energy‑Aware Balancing

Prefers low‑power devices (efficiency cores, integrated GPU) when @energy_aware active.

7. Determinism

Under --reproducible, PRNG seed fixed; device selection deterministic.

8. Error Handling

pub enum BalanceError {

    DeserializationFailed(Text), IncompatibleDevice,
}
