Blaze Phase 3a – Core Library: Sliding Windows

Previous: Units | Next: Deduplicate

Goal

Zero‑cost abstractions for fixed‑size overlapping/strided windows of slices, with SoA integration.

1. Core Traits

1.1 Windowable

pub trait Windowable<const N: usize> {
    type WindowIter: Iterator<Item = &[Self::Item; N]>;
    fn window(&self) -> Self::WindowIter;
}

1.2 SoAWindowable

pub trait SoAWindowable<const N: usize> {
    type WindowIter: Iterator<Item = WindowedView<Self, N>>;
    fn window_soa(&self) -> Self::WindowIter;
}

Automatically implemented for @layout(soa) structs.

2. Iterator Types

  • Window<T, const N: usize> – yields &[T; N].
  • WindowMut<T, const N: usize> – yields &mut [T; N].
  • StridedWindow<T, const N: usize, const S: usize> – stride S.
  • WindowSoA<S, const N: usize> – compiler‑generated for SoA.

All provide Iterator implementations with bounds checks in debug, elided in release.

3. WindowedView<S, N>

Compiler‑synthesized type for SoA. Provides per‑field slice access: pub fn field(&self) -> &[Type; N];

4. Window Hints

  • @slide(N) – optimise for window width.
  • @window_nobounds – omit bounds checks.
  • @window_accumulate – associative reduction, may replace with prefix‑sum.

5. Testing

  • Window: overlapping sequences.
  • WindowMut: mutate and verify original changed.
  • StridedWindow: various strides.
  • SoAWindowable: with @layout(soa) struct.
