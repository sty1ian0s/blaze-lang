Blaze Phase 3a – Core Library: Process Management

Previous: Operator Traits | Next: Units

Goal

Provide Command for spawning child processes, Output, and ExitStatus.

1. Command

pub struct Command {
    program: Text,
    args: Vec<Text>,
    env: Vec<(Text, Text)>,
    current_dir: Option<Text>,
}
impl Command {
    pub fn new(program: &str) -> Command;
    pub fn arg(&mut self, a: &str) -> &mut Command;
    pub fn env(&mut self, key: &str, val: &str) -> &mut Command;
    pub fn current_dir(&mut self, dir: &str) -> &mut Command;
    pub fn output(&mut self) -> Result<Output, Error>;
    pub fn status(&mut self) -> Result<ExitStatus, Error>;
}

2. Output

pub struct Output {
    pub status: ExitStatus,
    pub stdout: Vec<u8>,
    pub stderr: Vec<u8>,
}

3. ExitStatus

pub struct ExitStatus(i32);
impl ExitStatus {
    pub fn success(&self) -> bool;
    pub fn code(&self) -> Option<i32>;
}

4. Error

pub struct Error(i32);

Platform‑specific error code; may add message retrieval later.

5. Implementation Notes

  • Unix: fork/exec or posix_spawn.
  • Windows: CreateProcess.
  • Environment inherited unless overridden.
  • arg appends strings; program name is argv[0].

6. Testing

  • Execute echo (Unix) / cmd /c echo (Windows) – capture output.
  • Environment variable override.
  • Exit status for success and failure.
  • status() with no output.
