Blaze Phase 3b – Systems Library: Path Manipulation

Previous: OS | Next: Synchronization

Goal

Platform‑independent path manipulation via Path struct.

1. Path Struct

pub struct Path { inner: Text }

Linear; cloning requires explicit .clone().

1.1 Construction

pub fn new(s: &str) -> Path;

1.2 Accessors

pub fn as_str(&self) -> &str;

1.3 Joining

pub fn join(&self, other: &str) -> Path;

Uses platform‑specific separator; absolute paths replace.

1.4 Parent Directory

pub fn parent(&self) -> Option<Path>;

None for root or single component.

1.5 File Extension

pub fn extension(&self) -> Option<&str>;

Substring after last dot; excludes hidden files.

2. Platform‑Specific Behaviour

  • Separator: '/' (Unix) or '\\' (Windows).
  • Root detection: "/" (Unix) or "C:\" (Windows).
  • No automatic normalisation (.., .).

3. Trait Implementations

  • Clone – deep copy.
  • Dispose – drops inner Text.

4. Testing

  • Construction and as_str().
  • Join: correct separator; absolute path replacement.
  • Parent: "/foo/bar" → "/foo"; "foo" → None; root → None.
  • Extension: file with no extension, single dot, multiple dots, hidden.
  • Platform‑specific tests with @cfg.
