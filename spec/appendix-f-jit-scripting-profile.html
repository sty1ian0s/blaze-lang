Appendix F – JIT Scripting Profile (Expanded)

Previous: Appendix E | Next: Appendix G

Status

Normative. Defines relaxed rules for .blz files executed via blaze run or blaze repl, and blaze fix --aot-ify.

F.1 Activation

JIT scripting profile automatically applied to any file loaded by blaze run or blaze repl. Never applied to library code compiled by blaze build.

F.2 Top‑Level Statement Wrapping

Any expression or statement at top level not an item is automatically wrapped into a synthesised main function. If an explicit main exists, top‑level statements are a compile‑time error.

F.3 Extended Type Inference

  • Parameter types may be omitted (inferred from body).
  • Return types may be omitted (inferred as least upper bound).
  • Placeholder type _ for simple generic functions (e.g., fn identity(x: _) -> _ { x }).
  • where clauses may be omitted; compiler infers bounds from usage.

F.4 Implicit Global Region

Every script executes inside an implicit region block spanning the entire program execution. Default arena allocator freed on script exit.

F.5 Relaxed Effect Annotations

Explicit effect specifications (with io, with pure, etc.) are optional. If omitted, inferred set is used. If provided, must be a superset of inferred set.

F.6 Automatic Imports

Automatically imports: std::io::*, std::collections::*, std::iter::*, std::window::*, std::string::*, std::fs::*, std::env::*, std::process::*. Unused imports ignored; blaze fix removes them.

F.7 Implicit Cloning for @copy Types

Affine rules relaxed for @copy types; compiler silently duplicates values instead of requiring .clone(). move keyword forces move. 

let x = 5;
let y = x + x;   // allowed in JIT

blaze fix inserts @copy annotations or keeps affine.

F.8 Top‑Level Mutable Statics (@global)

@global let mut counter = 0;
fn increment() { counter += 1; println(counter); }

In JIT mode, allocated with safe Mutex (or atomic). In AOT mode, forbidden; blaze fix rewrites to AppState struct passed as &mut parameter or converted to actor.

F.9 Automatic Closure Captures

  • If variable used once and not by other closures → moved.
  • If used by multiple closures or beyond lifetime → wrapped in Rc/Arc and cloned transparently.

blaze fix rewrites with explicit capture clauses (move, clone, &).

F.10 Dynamic Dispatch by Default for Trait Objects

JIT may box values into dyn Trait and use dynamic dispatch. blaze fix monomorphises hot paths (guided by @hot or profiling).

F.11 Automatic Error Propagation

try! macro wraps expressions that may fail; prints error to stderr and terminates function/script. 

let data = try!(std::fs::read_to_text("config.toml"));

blaze fix replaces try! with ? and adds Result return types.

F.12 blaze fix --aot-ify

Converts JIT script to fully‑annotated AOT module:

  1. Add type annotations (parameters, return, generics, where).
  2. Add @copy annotations.
  3. Replace @global with AppState struct.
  4. Rewrite closures with explicit captures.
  5. Replace try! with ? and Result.
  6. Remove unused imports.
  7. Monomorphise hot paths (dyn Trait → impl Trait).
  8. Format output with blaze fmt.

Idempotent; can be run automatically in blaze.toml: 

[profile.release]
jit-to-aot = true

F.13 File Extension

Blaze source files use .blz. JIT profile applies when executed via blaze run or blaze repl. blaze build always uses AOT mode.
