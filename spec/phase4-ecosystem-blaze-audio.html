Blaze Phase 4 – Ecosystem: Audio

Previous: Async Std | Next: Backtrace

Goal

Provide low‑latency audio I/O, playback/recording, waveform generation, effects, and spatial audio integration with ECS.

1. Audio Devices

pub struct AudioDevice { name: Text, device_type: DeviceType, default_sample_rate: u32, num_channels: u16 }
pub enum DeviceType { Playback, Capture }
pub fn enumerate() -> Vec<AudioDevice>;

2. Streams

pub fn open_playback_stream(device: &AudioDevice, config: &StreamConfig, callback: impl FnMut(&mut [f32]) + Send + 'static) -> Result<AudioStream, AudioError>;
pub fn open_capture_stream(device: &AudioDevice, config: &StreamConfig, callback: impl FnMut(&[f32]) + Send + 'static) -> Result<AudioStream, AudioError>;

StreamConfig: sample rate, channels, buffer size. Callback must be pure (no I/O, no allocation).

3. Audio Buffer

pub struct AudioBuffer { samples: Vec<f32>, sample_rate: u32, channels: u16, duration: Duration }
pub fn load_wav(path: &str) -> Result<AudioBuffer, AudioError>;
pub fn save_wav(path: &str, buffer: &AudioBuffer) -> Result<(), AudioError>;

4. Waveform and Effects

pub enum Waveform { Sine, Square, Sawtooth, Triangle, Noise, Custom(fn(f32) -> f32) }
pub fn generate_waveform(wave: Waveform, frequency: f32, sample_rate: u32, duration: Duration) -> AudioBuffer;
pub trait AudioEffect { fn process(&mut self, buffer: &mut [f32], sample_rate: u32); }
pub struct Gain { factor: f32 } /* etc */

5. Spatial Audio

Integration with ECS: AudioEvent enum (PlayOneShot, SetListener, StopEmitter).

6. Error Handling

pub enum AudioError {
    Io(std::io::Error),
    DeviceUnavailable,
    FormatNotSupported,
    LoadError(Text),
    StreamClosed,
}

7. Testing

  • Playback: fill with sine, no errors.
  • Capture: record 1s, check length.
  • WAV round‑trip.
  • Effect chain: gain applied.
  • Spatial: position attenuation.
