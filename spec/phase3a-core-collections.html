Blaze Phase 3a – Core Library: Collections

Previous: Comparison | Next: Debug

Goal

Implement fundamental data structures: List<T> (doubly‑linked list), Map<K,V> (B‑tree), SlotMap<K,T> (generational arena), and RingBuf<T> (ring buffer). All respect linear ownership and implement Dispose.

1. List<T> – Doubly‑Linked List

pub struct List<T> { /* private */ }
impl<T> List<T> {
    pub fn new() -> Self;
    pub fn push_front(&mut self, value: T);
    pub fn push_back(&mut self, value: T);
    pub fn pop_front(&mut self) -> Option<T>;
    pub fn pop_back(&mut self) -> Option<T>;
    pub fn front(&self) -> Option<&T>;
    pub fn back(&self) -> Option<&T>;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
}
impl<T> Dispose for List<T> { ... }
impl<T> IntoIterator for List<T> { ... }

Nodes allocated in current region. Iterator consumes the list.

2. Map<K,V> – B‑Tree Map

pub struct Map<K, V> { /* private B‑tree fields */ }
impl<K: Ord, V> Map<K, V> {
    pub fn new() -> Self;
    pub fn insert(&mut self, key: K, value: V) -> Option<V>;
    pub fn get(&self, key: &K) -> Option<&V>;
    pub fn get_mut(&mut self, key: &K) -> Option<&mut V>;
    pub fn remove(&mut self, key: &K) -> Option<V>;
    pub fn contains_key(&self, key: &K) -> bool;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn iter(&self) -> MapIter<K, V>;
}
impl<K: Ord, V> Dispose for Map<K, V> { ... }
impl<K: Ord, V> IntoIterator for Map<K, V> { ... }

B‑tree with order 16 (configurable). Keys must be Ord.

Complexity verification: Insert/lookup O(log n); after 100k random inserts, height ≤5; iteration O(n) sorted.

3. SlotMap<K, T> – Generational Arena 

pub struct SlotMap<K, T> { /* private */ }
pub trait Key: Copy + Eq {}
impl<K: Key, T> SlotMap<K, T> {
    pub fn new() -> Self;
    pub fn insert(&mut self, value: T) -> K;
    pub fn get(&self, key: K) -> Option<&T>;
    pub fn get_mut(&mut self, key: K) -> Option<&mut T>;
    pub fn remove(&mut self, key: K) -> Option<T>;
    pub fn contains(&self, key: K) -> bool;
    pub fn len(&self) -> usize;
}
impl<K: Key, T> Dispose for SlotMap<K, T> { ... }

Keys are copyable and include generation; removals invalidate key (generation bump).

4. RingBuf<T> – Ring Buffer

pub struct RingBuf<T> { /* private */ }
impl<T> RingBuf<T> {
    pub fn new(capacity: usize) -> Self;
    pub fn push_back(&mut self, value: T) -> Result<(), T>;
    pub fn pop_front(&mut self) -> Option<T>;
    pub fn front(&self) -> Option<&T>;
    pub fn back(&self) -> Option<&T>;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn capacity(&self) -> usize;
}
impl<T> Dispose for RingBuf<T> { ... }

Fixed capacity; push_back returns Err(value) when full.

5. Testing

  • List: push/pop, iterator, disposal.
  • Map: insert/get/remove, iterator order, disposal.
  • SlotMap: insert/retrieve, key invalidation, disposal.
  • RingBuf: circular fill/overflow, front/back.
  • Complexity: B‑tree lookup O(log n), verify with 100k keys.
