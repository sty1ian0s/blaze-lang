Blaze Phase 4 – Ecosystem: Log

Previous: Layout | Next: LSP Extras

Goal

Structured, performant, data‑oriented logging framework with levels, key‑value pairs, compile‑time filtering, multiple backends (console, file, syslog, network), actor integration.

1. Log Levels

pub enum Level { Trace, Debug, Info, Warn, Error, Fatal }

2. log! Macro

log!(Info, "connection established", port: 443, protocol: "HTTPS");

Deferred formatting; LogRecord stores raw message and key‑value pairs.

3. LogRecord

pub struct LogRecord<'a> {
    timestamp: Instant,
    level: Level,
    target: &'a str,
    module: &'a str,
    line: u32,
    message: &'a str,
    fields: &'a [LogField<'a>],
}
pub struct LogField<'a> { key: &'a str, value: LogValue<'a> }
pub enum LogValue<'a> { Str(&'a str), Int(i64), Float(f64), Bool(bool), Text(&'a Text), Bytes(&'a [u8]), Error(&'a dyn Error), Debug(&'a dyn Debug), Display(&'a dyn Display) }

4. Logger Trait

pub trait Logger: Send + Sync + 'static { fn log(&self, record: &LogRecord); fn flush(&self); }

5. Built‑in Loggers

  • ConsoleLogger – coloured, human‑readable.
  • JsonLogger – NDJSON.
  • FileLogger – rotation.
  • SyslogLogger – Unix syslog.
  • NullLogger – discard.
  • CompositeLogger – fan‑out.

6. Configuration

pub struct LogConfig { level: Level, format: LogFormat, filters: Vec<LogFilter>, max_level_per_module: Map<Text, Level> }
pub enum LogFormat { Compact, Pretty, Json }
