Blaze Phase 3c – Extended Library: Regular Expressions

Previous: Random | Next: SIMD

Goal

Provide a regular expression engine (NFA/DFA) supporting common regex syntax with linear‑time search, avoiding catastrophic backtracking.

1. Regex Type

pub struct Regex { /* compiled representation */ }
impl Regex {
    pub fn new(pattern: &str) -> Result<Regex, Error>;
}

Linear; not cloneable.

2. Matching Methods

  • is_match(&self, text: &str) -> bool
  • find(&self, text: &str) -> Option<Match>
  • captures(&self, text: &str) -> Option<Captures>

3. Match Types

pub struct Match { start: usize, end: usize, text: Text }
impl Match { pub fn start(&self) -> usize; pub fn end(&self) -> usize; pub fn as_str(&self) -> &str; }
pub struct Captures { /* captures: Vec<Option<Match>> */ }
impl Captures { pub fn get(&self, index: usize) -> Option<Match>; pub fn len(&self) -> usize; }

4. Regex Syntax

Subset of Perl‑compatible: literals, classes ([abc], [a‑z], \d, \w, \s), repetition (*, +, ?, {n,m}, greedy/lazy), alternation (|), anchors (^, $), grouping ((...) capturing, (?:...) non‑capturing). No backreferences, look‑ahead, etc. (unsupported features produce error).

5. Error Handling

pub struct Error { message: Text, pos: usize }
impl Error { pub fn message(&self) -> &str; pub fn pos(&self) -> usize; }

6. Implementation Notes

  • Compiles pattern to NFA, optionally to DFA for forward scan.
  • Uses tagged NFA states for capture groups.
  • Linear‑time worst‑case (no exponential blow‑up).

7. Testing

  • Valid patterns: literal, classes, repetition, anchors, groups, alternation.
  • Invalid patterns: syntax error with position.
  • Capture groups: correct sub‑matches; non‑participating groups return None.
  • Performance: (a*)*b on 1000 'a' → linear time (test via timeout).
