Blaze Phase 4 – Ecosystem: LSP Extras

Previous: Log | Next: LZ4

Goal

Provide additional, composable Language Server Protocol (LSP) features for Blaze editors and IDEs: semantic highlighting, inlay hints, call hierarchy, workspace symbol search, and refactoring actions.

1. Feature Trait

pub trait Feature: Send + Sync {
    fn name(&self) -> &'static str;
    fn capabilities(&self) -> ServerCapabilities;
    fn register(&self, server: &mut LspServer);
}

2. Semantic Tokens

Extends LSP token types with Blaze‑specific: struct, enum, trait, type_alias, function, method, parameter, variable, field, mutable, linear, effect.

3. Inlay Hints

  • Type hints: let x = 42; → let x: i32 = 42;
  • Parameter hints: draw_rect(10, 20, 100, 200); → draw_rect(x:10, y:20, width:100, height:200);
  • Chaining hints: iterator intermediate types.

4. Call Hierarchy

Incoming/outgoing call graphs from def‑use chains and monomorphisation records.

5. Workspace Symbols

Incrementally updated index of public symbols with fuzzy matching.

6. Refactoring Actions

  • Rename (workspace‑wide).
  • Extract Function.
  • Inline Variable.
  • Convert to @derive.

7. Configuration

[lsp_extras]
semantic_tokens = true
inlay_hints = true
inlay_hints_show_types = true
inlay_hints_show_parameter_names = true
call_hierarchy = true
workspace_symbols = true
refactoring = true

8. Testing

  • Semantic tokens: token types and modifiers.
  • Inlay hints: inferred types and parameter names.
  • Call hierarchy: graph edges.
  • Rename: all call sites updated.
  • Extract function: new function inserted.
