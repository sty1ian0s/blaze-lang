Blaze Phase 4 – Ecosystem: Metrics

Previous: LZ4 | Next: MIDI

Goal

Low‑overhead metrics collection (counters, gauges, histograms, summaries) with export to Prometheus, InfluxDB, Graphite, and logging. O(1) recording, no allocation in hot path.

1. Metric Types

pub struct Counter { value: AtomicU64 }
impl Counter { pub fn new() -> Self; pub fn inc(&self, delta: u64); pub fn get(&self) -> u64; pub fn reset(&self); }
pub struct Gauge { value: AtomicI64 }
impl Gauge { pub fn new() -> Self; pub fn set(&self, value: i64); pub fn inc(&self, delta: i64); pub fn dec(&self, delta: i64); pub fn get(&self) -> i64; }
pub struct Histogram { /* buckets, sum, count */ }
impl Histogram { pub fn new(config: HistogramConfig) -> Self; pub fn observe(&self, value: f64); }
pub struct Summary { /* quantiles, sum, count */ }
impl Summary { pub fn new(config: SummaryConfig) -> Self; pub fn observe(&self, value: f64); }

2. Metrics Registry

pub struct MetricsRegistry { /* global OnceLock */ }
impl MetricsRegistry {
    pub fn register<M: Metric + 'static>(name: &str, metric: M) -> Result<(), MetricsError>;
    pub fn counter(name: &str) -> Option<&'static Counter>;
    pub fn gauge(name: &str) -> Option<&'static Gauge>;
    pub fn histogram(name: &str) -> Option<&'static Histogram>;
    pub fn summary(name: &str) -> Option<&'static Summary>;
    pub fn collect(&self) -> Vec<MetricSnapshot>;
}

3. Labels

Labeled metrics via registry.labeled_counter(name, labels).

4. Export Backends

pub trait MetricsRecorder: Send + Sync + 'static { fn record(&self, snapshot: &[MetricSnapshot]); }
pub struct PrometheusRecorder; pub struct InfluxDbRecorder; pub struct GraphiteRecorder; pub struct LogRecorder; pub struct CsvRecorder;

5. Actor Integration

pub actor MetricsActor {
    registry: MetricsRegistry,
    recorders: Vec<Box<dyn MetricsRecorder>>,
    push_interval: Duration,
}

6. Error Handling

pub enum MetricsError { DuplicateMetric, InvalidLabel, ExporterError(Text), Io(std::io::Error) }
