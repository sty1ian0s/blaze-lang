Blaze Phase 4 – Ecosystem: Doc Gen

Previous: Debug Info | Next: ECS

Goal

Documentation generator from /// and //! comments to static HTML with search, cross‑references, doctest integration.

1. Core Types

pub struct DocCrate { name: Text, version: Text, modules: Vec<DocModule>, readme: Option<Text> }
pub struct DocModule { name: Text, doc: Option<Text>, items: Vec<DocItem>, submodules: Vec<DocModule> }
pub enum DocItem { Function(DocFunction), Struct(DocStruct), Enum(DocEnum), Trait(DocTrait), TypeAlias(DocTypeAlias), Constant(DocConstant), Static(DocStatic), Module(DocModule) }
/* Each DocX has name, doc, fields/methods etc. */

2. Parsing

pub fn extract_docs(crate_path: &str) -> Result<DocCrate, DocGenError>;

Uses std::meta reflection to extract public items and doc comments. Markdown parsing (CommonMark). Code blocks extracted as doctests.

3. HTML Generation

pub fn render_html(doc: &DocCrate, config: &DocConfig) -> Result<DocOutput, DocGenError>;
pub struct DocConfig { output_dir: Text, title: Text, favicon: Option<Text>, include_source: bool, theme: DocTheme }
pub enum DocTheme { Light, Dark, Auto }
pub struct DocOutput { files: Vec<(Text, Vec<u8>)>, search_index: Vec<u8> }

4. Search Index

JSON index of items for client‑side search.

5. LSP Integration

blaze lsp uses extract_docs for hover completions.

6. Error Handling

pub enum DocGenError {
    Io(std::io::Error),
    MetadataNotFound,
    ParseError(Text),
}

7. Testing

  • Doc extraction.
  • Code examples extraction.
  • HTML rendering.
  • Search index.
