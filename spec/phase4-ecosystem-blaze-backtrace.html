Blaze Phase 4 – Ecosystem: Backtrace

Previous: Audio | Next: Bench Utils

Goal

Provide zero‑cost backtrace capture, symbolication, and formatting using DWARF/PDB.

1. Core Types

pub struct Backtrace { frames: Vec<Frame>, thread_name: Option<Text> }
pub struct Frame {
    pub instruction_pointer: usize,
    pub symbol_name: Option<Text>,
    pub file: Option<Text>,
    pub line: Option<u32>,
    pub column: Option<u32>,
}

2. Capture

impl Backtrace {
    pub fn capture() -> Backtrace;
    pub fn capture_skip(skip: usize) -> Backtrace;
}

Uses compiler‑generated unwinding tables (`.eh_frame` or PDB). Pure (no allocation except final Vec).

3. Symbolication

impl Backtrace {
    pub fn symbolicate(&mut self) -> Result<(), BacktraceError>;
    pub fn symbolicated(self) -> Backtrace;
}
pub fn set_debug_path(path: &str);

Uses DWARF debug info, `.blzdbg` file, or platform APIs (`dladdr`, `SymFromAddr`).

4. Formatting

impl Backtrace {
    pub fn to_string(&self) -> Text;
    pub fn to_string_pretty(&self) -> Text;
}

5. Integration

Provides #[panic_handler] attribute to print backtrace on panic.

6. Error Handling

pub enum BacktraceError {
    Io(std::io::Error),
    NoDebugInfo,
    UnsupportedPlatform,
    SymbolNotFound,
}

7. Testing

  • Capture: non‑zero frames.
  • Symbolication: test function name appears.
  • Pretty formatting contains expected strings.
  • No debug info: symbol name None.
  • Thread safety.
