Blaze Phase 4 – Ecosystem: Arch

Previous: Accessibility | Next: Async Std

Goal

Zero‑copy reading/writing of CPU architecture‑specific binary formats (ELF, PE, Mach‑O, raw binary).

1. Core Types

pub enum ObjectFile {
    Elf32(ElfFile32),
    Elf64(ElfFile64),
    Pe(PeFile),
    MachO(MachOFile),
    Raw(Vec<u8>),
}

ElfFile64 contains header, section headers, sections, program headers, segments. Similar for other formats.

2. Parsing

pub fn parse(data: &[u8]) -> Result<ObjectFile, ArchError>;

Auto‑detects format from magic bytes.

3. Writing

pub fn write(obj: &ObjectFile, path: &str) -> Result<(), ArchError>;
pub fn write_to_vec(obj: &ObjectFile) -> Result<Vec<u8>, ArchError>;

4. Error Handling

pub enum ArchError {
    Io(std::io::Error),
    InvalidFormat,
    UnsupportedArchitecture,
    UnsupportedVersion,
}

5. Testing

  • ELF round‑trip (byte‑for‑byte equality).
  • PE round‑trip.
  • Invalid input → InvalidFormat.
