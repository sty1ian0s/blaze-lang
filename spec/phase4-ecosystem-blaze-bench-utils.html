Blaze Phase 4 – Ecosystem: Bench Utils

Previous: Backtrace | Next: Chrono

Goal

Provide lightweight benchmarking framework with bench! macro, Bencher, black_box, and reporters.

1. bench! Macro

bench!(name: &str, iterations: u64, || { /* code */ });

Reports time per iteration via global reporter. Uses black_box to prevent optimisation.

2. Bencher (Manual)

pub struct Bencher { config: BenchConfig }
impl Bencher {
    pub fn new(config: BenchConfig) -> Bencher;
    pub fn iter<F>(&mut self, f: F) -> Duration where F: FnMut();
    pub fn iter_with_setup<F, S>(&mut self, setup: S, f: F) -> Duration where S: FnMut(), F: FnMut();
}

3. BenchConfig

pub struct BenchConfig {
    pub iterations: u64,
    pub warm_up: u64,
    pub min_time: Option<Duration>,
    pub max_time: Option<Duration>,
    pub report_format: ReportFormat,
}

4. black_box

pub fn black_box<T>(val: T) -> T;

Prevents compiler optimising away the value.

5. Reporters

pub trait BenchReporter: Send + Sync + 'static { fn report(&self, name: &str, duration: Duration, iterations: u64); }
pub struct ConsoleReporter; pub struct JsonReporter; pub struct CsvReporter; pub struct NullReporter;

Global reporter set via set_global_reporter.

6. Integration with @bench

@bench attribute generates code using Bencher. blaze bench discovers and runs them.

7. Determinism

Under --reproducible, PRNG seed is fixed.

8. Testing

  • Basic bench runs without error.
  • black_box prevents elimination.
  • Configuration respected.
