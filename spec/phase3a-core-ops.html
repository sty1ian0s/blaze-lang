Blaze Phase 3a – Core Library: Operator Traits

Previous: Iterators | Next: Process

Goal

Implement std::ops module with traits for operator overloading. Each trait maps to a language operator.

1. Binary Operator Traits

All define Output type and Rhs defaulting to Self.

  • Add – +
  • Sub – -
  • Mul – *
  • Div – / (division by zero panics for integers; infinity for floats)
  • Rem – %
  • BitAnd – & (bitwise AND)
  • BitOr – | (bitwise OR)
  • BitXor – ^ (bitwise XOR)
  • Shl – << (shift left)
  • Shr – >> (shift right; arithmetic for signed)

2. Unary Operator Traits

  • Neg – -
  • Not – ! (logical or bitwise complement)

3. Compound Assignment Traits

  • AddAssign, SubAssign, MulAssign, DivAssign, RemAssign
  • BitAndAssign, BitOrAssign, BitXorAssign, ShlAssign, ShrAssign

Compiler can synthesise from binary traits if not explicitly implemented.

4. Standard Implementations

All primitive types implement appropriate traits. User types must implement manually.

5. Testing

  • Primitive types: operators work as expected.
  • Custom struct (e.g., Vec2) implements Add; + works.
  • += works if AddAssign implemented; otherwise synthesised from Add.
  • Custom Output type (e.g., Add<Duration> for Instant) works.
