Blaze Phase 4 – Ecosystem: Glob

Previous: FFI | Next: gRPC

Goal

Provide pure‑Blaze library for matching file paths against glob patterns (wildcards) with support for *, ?, [abc], [a‑z], {a,b}, ** recursive matching.

1. Core Types

pub struct Pattern { text: Text, tokens: Vec<PatternToken> }
enum PatternToken {
    Char(char),
    AnyChar,
    AnySequence,
    RecursiveSequence,
    Class { chars: Vec<ClassChar>, negated: bool },
    Alternation(Vec<Pattern>),
    Literal(Text),
}

2. Compilation

impl Pattern { pub fn new(glob: &str) -> Result<Pattern, GlobError>; }

Parses glob string; supports escaping with \.

3. Matching

impl Pattern { pub fn matches(&self, path: &str) -> bool; pub fn matches_path(&self, path: &Path) -> bool; }

Case‑sensitive by default; case_insensitive() builder. Anchors: matches entire path; if no / matches base name.

4. Iteration (feature `fs`)

pub fn glob(pattern: &Pattern, root: &Path) -> Result<impl Iterator<Item = Path>, GlobError>;
pub struct GlobEntry { path: Path, is_dir: bool }

5. Error Handling

pub enum GlobError { Parse(Text), Io(std::io::Error) }

6. Testing

  • *.txt matches foo.txt.
  • [a‑z]*.txt matches abc.txt.
  • {*.rs,*.blz} matches both.
  • src/**/*.rs recursive.
  • Case‑insensitive flag.
  • File‑system iteration.
