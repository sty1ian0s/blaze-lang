Blaze Phase 3a – Core Library: Deduplicate

Previous: Sliding Windows | Next: Systems – Binary

Goal

Define Deduplicate trait and compile‑time deduplication mechanism for constant values.

1. Deduplicate Trait

pub trait Deduplicate {}

Marker trait. Signals compiler to globally deduplicate constants of this type.

Compiler: after monomorphisation, collects constants; if bit‑identical, emits one copy and redirects references. Recursively compares referenced data (Vec, Text, Owned).

Derivation

@derive(Deduplicate) struct MaterialParams { ... }

Available for struct, enum, union (union deduplicated by comparing all fields).

Limitations

  • Only constants (compile‑time known).
  • Per monomorphised instantiation for generics.
  • Size threshold (default 1 MB) – compiler may refuse to deduplicate larger types.

2. Compiler Implementation

Pass runs after monomorphisation and constant folding, before code generation.

  1. Collect constants of types with Deduplicate.
  2. Compute hash (e.g., SHA‑256) of each constant's data representation.
  3. Group by hash; verify bit‑identity.
  4. Emit a single copy per group; replace references with pointer to canonical copy.

Deterministic.

3. Integration with Asset Pipeline

blaze‑asset uses @derive(Deduplicate) for texture, mesh, material constants to eliminate duplicates across levels.

4. Testing

  • Two identical constants → one copy in binary (check via addresses or object file inspection).
  • Non‑identical constants → two copies.
  • Recursive deduplication (Vec, Text).
  • Large constants: threshold respected.
