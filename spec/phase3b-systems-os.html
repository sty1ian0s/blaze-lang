Blaze Phase 3b – Systems Library: Platform OS

Previous: Futures | Next: Path

Goal

Platform‑specific raw system call wrappers and types for Unix and Windows.

1. Module Structure

Parent module std::os with sub‑modules unix and windows, guarded by @cfg(target_os = ...).

2. std::os::unix

Re‑exports fundamental types: c_int, pid_t, uid_t, mode_t, off_t, size_t, ssize_t; constants: O_RDONLY, O_CREAT, S_IFMT, etc.; unsafe syscalls: open, close, read, write, lseek, fsync, ftruncate, stat, fstat, mkdir, unlink, rmdir.

stat struct with fields: st_dev, st_ino, st_mode, st_nlink, st_uid, st_gid, st_rdev, st_size, st_blksize, st_blocks, st_atime, st_mtime, st_ctime and nanosecond variants.

3. std::os::windows

Re‑exports: HANDLE, DWORD, LPVOID, LPCWSTR, BOOL; constants: GENERIC_READ, GENERIC_WRITE, CREATE_NEW, CREATE_ALWAYS, OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, INVALID_HANDLE_VALUE; unsafe syscalls: CreateFileW, ReadFile, WriteFile, CloseHandle.

Helper: fn to_utf16(s: &str) -> Vec<u16> for UTF‑8 to wide string.

4. Testing

  • Unix: open temp file, write via raw syscalls, read back.
  • Windows: similarly via Win32 API.
  • Compile‑time selection: code using std::os::unix fails on Windows, and vice versa.
  • to_utf16 tested.
