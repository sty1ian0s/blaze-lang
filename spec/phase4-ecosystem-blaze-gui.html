Blaze Phase 4 – Ecosystem: GUI

Previous: GPU Compute | Next: HTTP

Goal

High‑performance, data‑oriented, actor‑based GUI system with immediate‑mode rendering, constraint‑based layout, typed styles, automatic accessibility.

1. Widget Tree

pub enum WidgetNode {
    Container { id: WidgetId, child: WidgetHandle },
    Row { id: WidgetId, children: Vec<WidgetHandle> },
    Column { id: WidgetId, children: Vec<WidgetHandle> },
    Stack { id: WidgetId, children: Vec<WidgetHandle> },
    Text { id: WidgetId, content: String },
    Button { id: WidgetId, label: String, on_click: Message },
    Slider { id: WidgetId, value: f64, min: f64, max: f64, on_change: Message },
    Checkbox { id: WidgetId, checked: bool, label: String, on_toggle: Message },
    Image { id: WidgetId, source: ImageSource },
    Canvas { id: WidgetId, draw_fn: fn(&mut CanvasContext) },
    Custom { id: WidgetId, shape: Box<dyn CustomWidget> },
}

2. Layout

pub struct Layout { width: SizeConstraint, height: SizeConstraint, padding: EdgeInsets, margin: EdgeInsets, alignment: Alignment, flex: f64 }
pub enum SizeConstraint { Pixels(f64), Percent(f64), Auto }
fn compute_layout(tree: &[WidgetNode], available_width: f64, available_height: f64) -> Vec<Rect>;

3. Styling

pub struct Style { background: Color, foreground: Color, font_size: f64, font_family: String, corner_radius: f64, border: Option<(f64, Color)>, shadow: Option<(f64, f64, Color)> }

4. Accessibility

pub struct AccessibilityNode { role: AccessibilityRole, label: String, value: Option<String>, actions: Vec<AccessibilityAction>, bounding_rect: Rect }
pub enum AccessibilityRole { Button, Slider, CheckBox, StaticText, Image, List, ListItem, Heading, Window }

5. Input

Unified InputEvent handled by App actor; hit‑testing translates to messages.

6. Rendering

pub trait Renderer { fn draw_widget(&mut self, node: &WidgetNode, rect: Rect, style: &Style, state: &AppState); fn present(&mut self); }
