Blaze Phase 3a – Core Library: Formatting

Previous: Environment | Next: Hashing

Goal

Provide string formatting, the Display and Debug traits, and print/println functions.

1. Public Functions

pub fn print(args: Arguments);
pub fn println(args: Arguments);

Write formatted output to stdout; println adds newline. Panic on write failure.

2. Arguments Type

pub struct Arguments { /* opaque */ }

Produced by format! macro; compiler‑generated representation.

3. Formatter Type

pub struct Formatter { /* private */ }
impl Formatter {
    pub fn write_str(&mut self, s: &str) -> Result<(), Error>;
    pub fn write_fmt(&mut self, args: Arguments) -> Result<(), Error>;
}

Accumulates output into a buffer; write_fmt formats arguments.

4. Error Type

pub type Error = ();

Infallible formatting; always Ok(()).

5. Traits

5.1 Display

pub trait Display {
    fn fmt(&self, f: &mut Formatter) -> Result<(), Error>;
}

5.2 Debug

pub trait Debug {
    fn fmt(&self, f: &mut Formatter) -> Result<(), Error>;
}

@derive(Display) and @derive(Debug) generate implementations.

6. Implementation

Formatter uses a Vec<u8> buffer. print/println create a Formatter, call args.write_fmt, then write buffer to stdout.

7. Testing

  • print/println output captured.
  • Display and Debug for built‑ins produce expected strings.
  • Manual implementations for test structs.
  • All formatting returns Ok(()).
