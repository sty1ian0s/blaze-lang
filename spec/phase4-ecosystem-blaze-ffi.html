Blaze Phase 4 – Ecosystem: FFI

Previous: ESP32 | Next: Glob

Goal

Safe FFI builder: #[ffi_export] and #[ffi_import] attributes, automatic generation of C headers and shared library exports, CType trait, blaze‑ffi‑tool.

1. Attributes

#[ffi_export] fn add(a: i32, b: i32) -> i32 { a + b }
#[ffi_export] #[repr(C)] struct Point { x: f64, y: f64 }
#[ffi_export] enum Status { Ok, Error }

Generates C wrappers, headers, and shared library exports.

2. Import

#[ffi_import] extern "C" { fn malloc(size: usize) -> *mut u8; fn free(ptr: *mut u8); }

3. CType Trait

pub trait CType { type CEquivalent; fn to_c(self) -> Self::CEquivalent; unsafe fn from_c(c: Self::CEquivalent) -> Self; }

4. blaze‑ffi‑tool

blaze ffi --crate mylib --out-dir ffi/

5. Safety Guarantees

Linear types transferred across FFI must be consumed; references require explicit lifetimes.

6. Error Handling

pub enum FfiError {
    Io(std::io::Error),
    UnsupportedType(Text),
    LayoutMismatch(Text),
    LinkError(Text),
}

7. Testing

  • Export add function, call from C.
  • Export struct, call from C.
  • Import C function (abs).
  • Reject unsound signatures.
