Blaze Phase 4 – Ecosystem: CLI

Previous: Chrono | Next: Compress

Goal

Declarative CLI argument parsing using @cli macro, subcommands, help generation, integration with blaze‑config.

1. @cli Macro

@cli(name = "myapp", version = "1.0", about = "Does wonderful things")
struct App {
    host: String,
    port: u16,
    #[short('v')] verbose: bool,
    #[long("config-file")] config_file: Option<String>,
    #[subcommand] command: Option<Command>,
}

Generates CliParser and CliHelp.

Field attributes: #[long], #[short], #[default], #[arg_name], #[help], #[required], #[multiple], #[conflicts_with], #[requires].

2. Subcommands

Represented by enum where each variant is a command. Nested subcommands supported.

3. Parsing and Execution

pub struct CliParser<T: Cli> { /* … */ }
impl<T: Cli> CliParser<T> {
    pub fn new() -> Self;
    pub fn parse(self) -> Result<T, CliError>;
    pub fn parse_from(self, args: &[Text]) -> Result<T, CliError>;
    pub fn parse_with_config(self, config: ConfigLoader<T>) -> Result<T, CliError>;
    pub fn print_help(&self);
    pub fn print_version(&self);
}

4. Integration with blaze‑config

Loading order: defaults → config file → environment → CLI arguments.

5. Help Generation

Auto‑generated from struct fields, doc comments, attributes; 80‑column width; optional colours.

6. Error Handling

pub enum CliError {
    MissingRequiredOption(Text),
    InvalidValue(Text),
    UnrecognizedOption(Text),
    MissingSubcommand,
    InvalidSubcommand(Text),
    Io(std::io::Error),
    ConfigError(ConfigError),
}

7. Type Support

Strings, integers, floats, booleans, durations, paths, enums, Option, Vec. Custom types via FromStr.

8. Testing

  • Basic parsing.

  • Conflicts/requires.
  • Help output.
  • Integration with config.
