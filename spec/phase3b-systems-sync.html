Goal

Provide channel types (Sender, Receiver) and Capability<A> for actor message passing.

1. Sender<T>

pub struct Sender<T> { /* reference‑counted pointer to inner queue */ }
impl<T> Sender<T> {
    pub fn send(&self, value: T) -> Result<(), SendError<T>>;
}
impl<T> Clone for Sender<T> { ... }
impl<T> Dispose for Sender<T> { ... }

Unbounded channel (default). send returns SendError if receiver dropped.

2. Receiver<T>

pub struct Receiver<T> { /* owns receiving end */ }
impl<T> Receiver<T> {
    pub fn recv(&mut self) -> Option<T>;
}
impl<T> Iterator for Receiver<T> { type Item = T; fn next(&mut self) -> Option<T>; }
impl<T> Dispose for Receiver<T> { ... }

Linear; recv blocks? Actually non‑blocking: returns None if empty and all senders gone. For actors, we use async channels.

3. Channel Creation

pub fn channel<T>() -> (Sender<T>, Receiver<T>);
pub fn channel_with_capacity<T>(capacity: usize) -> (Sender<T>, Receiver<T>);

Bounded channel optional; sender blocks when full.

4. SendError<T>

pub struct SendError<T>(pub T);

5. Capability<A>

pub struct Capability<A> { /* opaque handle */ }
impl<A> Clone for Capability<A> { ... }
impl<A> Dispose for Capability<A> { ... }

Created by spawn(). Compiler generates message‑sending methods on this type.

6. Implementation Notes

  • Channels use lock‑free linked list or ring buffer with atomics.
  • Capability holds pointer to actor's mailbox and supervisor reference.

7. Testing

  • Channel: send/receive, clone sender, drop receiver → send error.
  • Capability: clone and send via actor.
  • Dispose: sender drop closes; receiver drop discards pending messages.
