Blaze Phase 5 – Ecosystem: Plot

Previous: Tensor | Next: CUDA

Goal

Data‑oriented, declarative plotting library that integrates seamlessly with blaze‑gui. Plots are created by composing mark layers on a Plot builder, and the final result is a WidgetHandle that can be placed directly inside any blaze‑gui layout. All rendering is pure (or carries the gpu effect when a GPU backend is used), and the plot is automatically accessible and responsive.

1. Core Concepts

A plot is a pure function that takes data and configuration and returns a WidgetHandle. Internally, it is a WidgetNode::Canvas with a specialised draw function. The plot participates in layout, accessibility, and event handling automatically.

2. Plot Builder API

pub struct Plot {
    layers: Vec<Layer>,
    width: f64,
    height: f64,
    title: Option<Text>,
    x_axis_label: Option<Text>,
    y_axis_label: Option<Text>,
    color_scale: ColorScale,
    legend: Option<LegendConfig>,
    margin: EdgeInsets,
    background: Color,
}
impl Plot {
    pub fn new() -> Self;
    pub fn width(mut self, w: f64) -> Self;
    pub fn height(mut self, h: f64) -> Self;
    pub fn title(mut self, title: &str) -> Self;
    pub fn x_axis_label(mut self, label: &str) -> Self;
    pub fn y_axis_label(mut self, label: &str) -> Self;
    pub fn color_scale(mut self, scale: ColorScale) -> Self;
    pub fn legend(mut self, config: LegendConfig) -> Self;
    pub fn margin(mut self, margin: EdgeInsets) -> Self;
    pub fn background(mut self, color: Color) -> Self;
    pub fn layer(mut self, mark: impl IntoLayer) -> Self;
    pub fn handlem(self) -> WidgetHandle;
}

Marks: scatter, line, bar, histogram, heatmap, contour, text, error_bar, area, box_plot.

3. Integration with blaze‑gui

  • Layout: Respects width/height; Auto fills parent.
  • Styling: Uses parent Style plus plot‑specific properties.
  • Responsiveness: Automatically adapts to available width.
  • Interactivity: Built‑in zoom/pan; custom click handlers via wrapping.

4. Accessibility

Exposes StaticText role; title used as accessible name; data summarised for screen readers. Override with a11y method.

5. Rendering and Export

Plot can be rendered as a widget or exported to PNG/SVG via to_png and to_svg methods (headless).

6. Testing

  • Widget integration: builder → layout solver → rectangle.
  • Accessibility: role and name.
  • Export: PNG generation and comparison.
  • Event handling: scroll zoom.
