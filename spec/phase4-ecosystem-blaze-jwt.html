Blaze Phase 4 – Ecosystem: JWT

Previous: JSON | Next: Kafka

Goal

Pure, zero‑copy JSON Web Token (JWT) implementation with HMAC, RSA, ECDSA. Supports header, claims, signing, verification, validation (exp, nbf, iss, aud).

1. Core Types

pub struct Token { pub header: Header, pub claims: Claims, pub signature: Vec<u8> }
pub struct Header { pub algorithm: Algorithm, pub type_: Option<Text>, pub kid: Option<Text>, pub extra: Map<Text, JsonValue> }
pub struct Claims { pub issuer: Option<Text>, pub subject: Option<Text>, pub audience: Option<Text>, pub expiration: Option<u64>, pub not_before: Option<u64>, pub issued_at: Option<u64>, pub jwt_id: Option<Text>, pub extra: Map<Text, JsonValue> }
pub struct SignedToken { pub encoded: Text }

2. Signing

pub enum SigningKey { Hmac(Vec<u8>), Rsa(RsaPrivateKey), Ec(EcPrivateKey) }
pub fn sign(token: &Token, key: &SigningKey) -> Result<SignedToken, JwtError>;

3. Verification

pub enum VerificationKey { Hmac(Vec<u8>), Rsa(RsaPublicKey), Ec(EcPublicKey), None }
pub fn decode_and_verify(token: &SignedToken, key: &VerificationKey) -> Result<Token, JwtError>;
pub fn decode_without_verification(token: &SignedToken) -> Result<Token, JwtError>;

4. Validation

pub struct TokenValidator {
    pub validate_exp: bool,
    pub validate_nbf: bool,
    pub validate_iss: Option<Text>,
    pub validate_aud: Option<Text>,
    pub leeway: u64,
    pub custom_validator: Option<fn(&Claims) -> bool>,
}

5. Error Handling

pub enum JwtError {
    InvalidToken, InvalidHeader, InvalidSignature, InvalidKey,
    TokenExpired, TokenNotYetValid, InvalidIssuer, InvalidAudience,
    CustomValidationFailed, Serialization(Text), Base64(Text), Crypto(CryptoError),
}

6. Testing

  • Round‑trip for each algorithm.
  • Expiration validation.
  • Issuer/audience validation.
  • None algorithm rejection.
  • Invalid signature.
