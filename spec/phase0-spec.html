Blaze Phase 0 – Minimal Language Specification

Goal

Implement a Rust‑based interpreter (Rust 1.60.0) that can execute a tiny subset of Blaze. Later phases will add generics, effects, actors, etc.

1. Lexical Structure

1.1 Source Text

Unicode text, UTF‑8. Line breaks: LF or CRLF.

1.2 Keywords

In Phase 0, the interpreter only needs to recognise: fn if else while let return struct true false loop continue break.

1.3 Identifiers

Sequence of letters, digits, underscores (start with letter or _).

1.4 Literals

Integer literals (decimal digits only), boolean literals.

1.5 Operators

+ - * / % = == != < > <= >= && || ! ( ) { } ; , -> .

1.6 Comments

Single‑line //.

2. Syntax (EBNF – Phase 0)

program   = { item } ;
item      = fn_decl | struct_decl ;

fn_decl   = "fn" ident "(" param_list ")" block ;
param_list= [ ident { "," ident } ] ;
block     = "{" { stmt } "}" ;

struct_decl = "struct" ident "{" { ident ":" primitive_type ";" } "}" ;
primitive_type = "i32" | "bool" ;

stmt      = let_stmt | expr_stmt | if_stmt | while_stmt | loop_stmt
           | return_stmt | break_stmt | continue_stmt | block ;
let_stmt  = "let" ident "=" expr ";" ;
expr_stmt = expr ";" ;
if_stmt   = "if" expr block [ "else" ( if_stmt | block ) ] ;
while_stmt= "while" expr block ;
loop_stmt = "loop" block ;
break_stmt= "break" ";" ;
continue_stmt= "continue" ";" ;
return_stmt= "return" [ expr ] ";" ;

expr       = logical_or ;
logical_or = logical_and { "||" logical_and } ;
logical_and= equality { "&&" equality } ;
equality   = relational { ("==" | "!=") relational } ;
relational = additive { ("<" | ">" | "<=" | ">=") additive } ;
additive   = multiplicative { ("+" | "-") multiplicative } ;
multiplicative = unary { ("*" | "/" | "%") unary } ;
unary      = "-" unary | "!" unary | primary ;
primary    = literal | ident [ "(" args ")" ] | "(" expr ")" | struct_init | field_access ;
args       = [ expr { "," expr } ] ;
struct_init= ident "{" field_init { "," field_init } "}" ;
field_init = ident ":" expr ;
field_access= expr "." ident ;

3. Semantics (Phase 0)

  • Types: only i32 and bool.
  • Variables are affine (move semantics).
  • Arithmetic overflow panics; division by zero panics.
  • Control flow as usual.
  • Function arguments passed by value (move).
  • Recursion allowed.

4. Testing Requirements

Tests for lexer, parser, semantic checks, interpreter, overflow panics.
