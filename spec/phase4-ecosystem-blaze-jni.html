Blaze Phase 4 – Ecosystem: JNI

Previous: Input | Next: JSON

Goal

Safe, data‑oriented interface for calling Java code and writing Java native methods in Blaze. Wraps JNI with linear types.

1. Core Types

pub struct Jvm { /* … */ }
pub struct JClass { /* … */ }
pub struct JObject { /* … */ }
pub enum JValue {
    Bool(bool), Byte(i8), Char(u16), Short(i16), Int(i32), Long(i64), Float(f32), Double(f64), Object(JObject), Null,
}

2. Jvm Operations

impl Jvm {
    pub fn new() -> Result<Self, JniError>;
    pub fn attach_current_thread() -> Result<Self, JniError>;
    pub fn find_class(&self, name: &str) -> Result<JClass, JniError>;
    pub fn new_object(&self, class: &JClass, constructor_sig: &str, args: &[JValue]) -> Result<JObject, JniError>;
    pub fn call_method(&self, obj: &JObject, name: &str, sig: &str, args: &[JValue]) -> Result<JValue, JniError>;
    pub fn call_static_method(&self, class: &JClass, name: &str, sig: &str, args: &[JValue]) -> Result<JValue, JniError>;
}

3. Native Methods

#[jni_export]
fn Java_com_example_MyClass_add(env: &JniEnv, _class: &JClass, a: i32, b: i32) -> i32 {
    a + b
}

4. Error Handling

pub enum JniError {
    JvmInitFailed, ClassNotFound, MethodNotFound, FieldNotFound,
    ExceptionThrown(Text), InvalidArgument, Io(std::io::Error),
}

5. Testing

  • Create JVM, call System.getProperty("java.version").
  • Call java.lang.Math.abs(-1).
  • Native method exported and called from Java.
