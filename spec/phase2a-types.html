Blaze Phase 2a – Expanded Type System & Linear Ownership

Previous: Phase 1 | Next: Phase 2b – Generics

Goal

Extend the compiler to support all primitive types, compound types, structs, enums, unions, and the full linear type system (move semantics, partial moves, Dispose, @copy).

1. Primitive Types

1.1 Integer Types

  • Signed: i8, i16, i32, i64, i128, isize
  • Unsigned: u8, u16, u32, u64, u128, usize
  • Literals default to i32; suffixes select type.
  • Operations: arithmetic, bitwise, comparison. Overflow panics in debug, wraps in release.

1.2 Floating‑Point Types

  • f16, f32, f64, f128 (IEEE‑754).
  • Literals with decimal point default to f64.
  • Division by zero returns infinity; =~ for approximate equality.

1.3 Boolean, Character, String, Unit

  • bool with true/false.
  • char – Unicode scalar.
  • str – UTF‑8 slice, always behind reference.
  • () – unit type.

2. Compound Types

2.1 Tuples

(T1, T2, ...) – destructuring only, no positional indexing.

2.2 Arrays

[T; N] – fixed size, compile‑time constant N.

2.3 Slices

[T] – dynamically sized, used behind references.

2.4 References

  • &T – shared, implicitly copyable.
  • &mut T – exclusive, warning on aliasing.

2.5 Raw Pointers

*T, *mut T – unsafe, no automatic disposal.

3. Algebraic Data Types

3.1 Structs

Named product types with fields, optional default values. Construction: Name { field: val }.

3.2 Enums

Tagged unions with variants. match for pattern matching. Discriminants can be explicit.

3.2.1 @pack Enum Attribute

@pack(N) stores discriminant in exactly N bits. Example: @pack(6) enum WeaponType { Knife, Pistol, ... }.

3.3 Unions

union – unsafe field overlap, no automatic discriminant.

4. Linear Type System

All user‑defined types are linear by default (move semantics).

4.1 Move Semantics

Assignment, argument passing, destructuring move ownership. Source becomes uninitialized.

4.2 Partial Moves

Moving a single field leaves the rest valid. Struct cannot be used until all moved fields are reinitialized.

4.3 Dispose Trait

trait Dispose { fn dispose(&mut self); } – compiler inserts calls at scope exit. Primitive types and references are no‑op. Structs automatically implement if all fields do.

4.4 @copy Attribute

Marks type as copyable if size ≤ 16 bytes, all fields @copy, no owned pointers (raw pointers allowed).

5. Constants and Statics

5.1 const and static

const compile‑time constant; static global variable. @shared deduplicates across compilation units.

5.2 @compressed Static Attribute

@compressed(level = L, algorithm = "algo") compresses static data at build time. Runtime access returns decompressed view.

6. Type Checking & Inference

Full type checking; signatures fully annotated; local variables may be inferred.

7. Code Generation

Primitives map to C types. Structs become C structs; enums tag+union; @pack enums use bitfields. Moves zero/poison source; dispose inserted at scope exit.

8. Layout Attributes

  • @layout(auto) – default, hardware‑aware if std::hardware available.
  • @layout(soa) – Structure of Arrays.
  • @layout(aosoa, N = 8) – Array of Structure of Arrays with chunk size N (power of two, 1‑64).
  • @layout(packed) – bit‑packed fields.
  • @layout(C) – C‑compatible.
  • @layout(adaptive) – profile‑guided adaptive layout (see Appendix G).

9. Constrained Types (Negative Space)

Types may be refined with constraints evaluated at compile time.

9.1 Syntax

type_refinement = base_type [ "@" constraint ]
constraint      = "range" "(" const_expr "," const_expr ")"
                | "pattern" "(" string_lit ")"
                | "size" "(" const_expr ")"
                | "not" constraint
                | "and" "(" constraint "," constraint ")"
                | "or" "(" constraint "," constraint ")"
                | path

9.2 Semantics

Constraints are checked at compile time; if unprovable, runtime checks are inserted unless @assume or unsafe is used.

9.3 Examples

let x: i32 @range(1, 100) = 50;
let s: str @pattern(r"^[A-Z][a-z]*$") = "Hello";

10. Hardware Control Attributes

  • @cache_line(align = 64, hint = "prefetch")
  • @aligned(16)
  • @volatile
  • @non_temporal
  • @simd_lane(0..3)
  • @vectorize(enable = true, width = 8)
  • @masked
  • @device("gpu", kernel = "my_kernel")
  • @sync_with("gpu")
  • @atomic(ordering = "release")
  • @fence(scope = "device")

11. Testing

  • Positive/negative tests for each feature.
  • Partial moves and reinitialisation.
  • @copy acceptance/rejection.
  • Dispose enforcement.
  • Constraint solver tests: compile‑time proof, runtime fallback.
  • Hardware attributes: alignment, SIMD, device offloading.
  • Layout tests: SoA, AoSoA, adaptive with profiling.
