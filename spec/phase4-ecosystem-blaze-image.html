Blaze Phase 4 – Ecosystem: Image

Previous: I18n | Next: Input

Goal

Data‑oriented, zero‑copy image processing library with support for common pixel formats, colour space conversions, geometric transformations, filtering, and integration with blaze‑tensor and blaze‑serde. All functions are pure and automatically parallelised.

1. Core Types

pub struct Image {
    width: u32,
    height: u32,
    format: PixelFormat,
    color_space: ColorSpace,
    data: Vec<u8>,
}

PixelFormat: Gray8, Gray16, RGB8, RGBA8, BGRA8, RGB16, RGBA16, RGBA32F, BayerRG8, etc. ColorSpace: sRGB, Linear, CIELab, HSV, YCbCr. 

impl Image {
    pub fn new(width: u32, height: u32, format: PixelFormat, color_space: ColorSpace) -> Self;
    pub fn from_bytes(width: u32, height: u32, format: PixelFormat, color_space: ColorSpace, data: Vec<u8>) -> Self;
    pub fn from_raw(width: u32, height: u32, format: PixelFormat, color_space: ColorSpace, data: Vec<u8>) -> Result<Self, ImageError>;
    pub fn from_file(path: &str) -> Result<Self, ImageError>;
    pub fn from_memory(buf: &[u8], format: ImageFormat) -> Result<Self, ImageError>;
    pub fn width(&self) -> u32;
    pub fn height(&self) -> u32;
    pub fn format(&self) -> PixelFormat;
    pub fn color_space(&self) -> ColorSpace;
    pub fn as_bytes(&self) -> &[u8];
    pub fn as_mut_bytes(&mut self) -> &mut [u8];
    pub fn pixel_count(&self) -> usize;
}

2. Pixel Access

pub enum Pixel {
    Gray8(u8),
    Gray16(u16),
    RGB8(u8, u8, u8),
    RGBA8(u8, u8, u8, u8),
    RGBA16(u16, u16, u16, u16),
    RGBA32F(f32, f32, f32, f32),
}
impl Image {
    pub fn get_pixel(&self, x: u32, y: u32) -> Option<Pixel>;
    pub fn set_pixel(&mut self, x: u32, y: u32, pixel: &Pixel) -> Result<(), ImageError>;
    pub unsafe fn get_pixel_unchecked(&self, x: u32, y: u32) -> Pixel;
    pub unsafe fn set_pixel_unchecked(&mut self, x: u32, y: u32, pixel: &Pixel);
}
