Appendix G – Compiler Algorithm Details

Previous: Appendix F | Next: Appendix H

Status

Normative. Exact algorithms and data structures for semantic analysis and optimisation passes.

G.1 Linearity Checking

Operates per function body. Maintains Live (initialised variables) and Partial (moved fields). Process statements in source order.

G.1.1 Scalar Variables

  • Read: if in Live and Partial empty → valid, remove from Live.
  • Assignment: if not in Live → add; if in Live and Partial empty → error.
  • Binding: add to Live.
  • Argument passing: same as read.
  • End of scope: dispose all Live variables implementing Dispose; error for linear variables without Dispose.

G.1.2 Struct Fields

  • Read field: if live and not partially moved → add to Partial; if all fields moved → remove struct from Live.
  • Assign field: remove from Partial; if empty → struct fully live.
  • Whole‑struct read: remove from Live.
  • Whole‑struct assignment: add to Live.

G.1.3 Branches

For if/else, compute live/partial after each branch; merge: live if live in both with identical partial sets; dead otherwise. For if without else, conservative merge.

G.1.4 Loops

Fixed‑point iteration: start with live_start = L_in, compute live_end, repeat until stable. Variables moved inside loop considered dead after loop.

G.1.5 Function Calls and Returns

Arguments consumed; return value is new binding. Return consumes expression; all live variables disposed unless moved into return.

G.2 Effect Inference

G.2.1 Effect Lattice

Power set of {alloc, io, parallel, gpu, hal} with implications: io ⇒ alloc, gpu ⇒ parallel, hal ⇒ io.

G.2.2 Intra‑procedural

Initialize effect set to ∅; traverse body; add alloc for allocations, io for I/O, parallel for pure for loops. Store final set.

G.2.3 Fixed‑Point (Inter‑procedural)

Worklist algorithm: recompute each function's effect set using current callee sets; propagate until no changes. Termination guaranteed (lattice height ≤5).

G.3 @comptime Interpreter

Stack machine executing bytecode of @comptime functions. Deterministic. Instruction set includes PUSH_LIT, PUSH_VAR, ADD, CALL, REFLECT, etc. (see spec).

G.4 Optimisation Passes

G.4.1 Canonicalisation

DCE, constant folding, GVN, PRE, LICM, strength reduction, induction variable optimisation.

G.4.2 Specialisation (opt‑level ≥ 1)

Inlining, auto‑vectorisation, SoA transformation, auto‑parallelisation, allocation elision, region fusion, tail‑call optimisation, devirtualisation.

G.4.3 Code‑Gen Preparation

Bounds check elision, peephole, prefetch insertion, alignment, non‑temporal stores, loop versioning, redundant store elimination, idiom recognition.

G.5 Region Lifetime Inference (NLL)

Borrow point and use point tracking; borrow live from creation to last use; error if borrow outlives region. &mut exclusivity warning on overlapping live ranges.

G.6 Array Bounds Check Elision

Track value range of index via lattice; if proven within [0, len-1], remove bounds check.

G.7 Actor Runtime Scheduling

Work‑stealing thread pool (one worker per core). Local deque per worker; SPSC mailbox upgraded to MPSC on Capability clone. Actor runs until await or handler complete. Supervisor escalation: >3 panics in 10s escalates to parent.

G.8 @derive Expansion

Syntactic expansion for Debug, PartialEq, Default, Clone, Binary, Display, From/Into, AsRef/AsMut, Error, FromRow, Deduplicate, Refine.

G.9 .blzlib Format

.tar archive with three files: metadata.json (version, target, opt_level, modules), object.o (ELF/COFF), debug.blzdbg (BDF debug info).

G.10 Incremental Compilation

Fingerprint per source file; dependency graph of items; dirty set recomputed on change; unchanged AST reused.

G.11 Symbol Table Implementation

Index‑based arena with perfect hashing for global scope; local scopes use linear search or simple hash table.

G.12 Typed Intermediate Representation (TIR)

SoA storage: opcodes, dst, src1, src2, imm_u64, block_id. Basic blocks with preds/succs.

G.13 Adaptive Layout Compilation

For @layout(adaptive): instrumentation records access patterns; profiling run writes .blaze.prof; recompile selects optimal layout (soa, aosoa, auto).
