Blaze Phase 4 – Ecosystem: gRPC

Previous: Glob | Next: GPU Compute

Goal

gRPC client and server built on blaze‑http2, blaze‑serde, and Protocol Buffers. Supports unary, client‑stream, server‑stream, bidi‑streaming.

1. Core Concepts

  • MethodDescriptor – method metadata.
  • ServiceDescriptor – service metadata.
  • Channel – connection to gRPC server.
  • Server – gRPC server.

2. Protocol Buffers Integration

pub trait Message: Serialize + Deserialize<'static> + Send + Sync + 'static {
    fn descriptor() -> MessageDescriptor;
}

5. Error Handling

pub enum GrpcError { Status(Status), Transport(HttpError), Codec(Text) }

pub enum Code { Ok, Cancelled, Unknown, InvalidArgument, DeadlineExceeded, NotFound, AlreadyExists, PermissionDenied, ResourceExhausted, FailedPrecondition, Aborted, OutOfRange, Unimplemented, Internal, Unavailable, DataLoss, Unauthenticated }
