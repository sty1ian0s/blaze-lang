Blaze Phase 3c – Extended Library: Fixed‑Point Arithmetic

Previous: Dynamic | Next: Meta

Goal

Fixed‑point numeric types with compile‑time integer and fractional widths.

1. Fixed Type

pub struct Fixed<const N: usize, const M: usize> { bits: iN }

N total bits (8, 16, 32, 64, 128); M fractional bits (0 ≤ M < N). iN is the smallest signed integer holding N bits.

Constructors

pub fn from_raw(raw: iN) -> Self;
pub fn from_float(val: f64) -> Self;
pub fn from_int(val: i32) -> Self;
pub fn to_float(self) -> f64;
pub fn to_int(self) -> i32;
pub fn raw(self) -> iN;

from_float rounds to nearest ties‑to‑even; saturates.

2. Arithmetic Operations

Implements all std::ops traits: Add, Sub, Mul, Div, Rem, Neg, and assignment variants.

  • Add/Sub: direct integer addition/subtraction.
  • Mul: (a * b) >> M with widening to avoid overflow.
  • Div: (a << M) / b.
  • Rem: raw remainder.

Overflow: panic in debug, wrap in release; saturation methods available.

3. Saturation Methods

pub fn saturating_add(self, rhs: Self) -> Self;
pub fn saturating_sub(self, rhs: Self) -> Self;
pub fn saturating_mul(self, rhs: Self) -> Self;
pub fn saturating_neg(self) -> Self;

4. Comparisons, Bitwise, Traits

  • PartialEq, Eq, PartialOrd, Ord (raw bits).
  • Bitwise ops: &, |, ^, <<, >> (raw bits; radix point not preserved for shifts).
  • @copy, Clone, Default, Debug, Display, Hash.

5. Testing

  • Construction from float/int; raw bits.
  • Arithmetic compared to float equivalents (within epsilon).
  • Overflow and saturation.
  • Comparisons.
  • Precision: Fixed<8,4> step size 1/16.
