Blaze Phase 3c – Extended Library: SIMD

Previous: Regex | Next: GPU

Goal

Provide architecture‑specific SIMD vector types and operations (128‑bit, 256‑bit, 512‑bit) for high‑performance parallel computation. Fallback to scalar on unsupported platforms.

1. Vector Types

Examples: i8x16, i16x8, i32x4, i64x2, u8x16, f32x4, f64x2; similarly for 256‑bit (i8x32, f32x8) and 512‑bit (i8x64, f32x16). All are @copy.

2. Operations

All vectors support element‑wise arithmetic, bitwise, and comparison operators via std::ops.

Inherent methods: splat, new, extract, replace, load_unaligned, store_unaligned. For floats: abs, min, max, sqrt, floor, ceil, round, trunc.

3. Compiler Intrinsics

Re‑exports unsafe intrinsics for shuffles, permutations, reductions (guarded by @cfg(target_feature)).

4. Feature Detection

  • is_x86_feature_detected(feature: &str) -> bool
  • is_arm_feature_detected(feature: &str) -> bool
  • @cfg(target_feature = "avx2") for compile‑time.

5. Fallback

On unsupported targets, vector types are tuples of scalars; operations use element‑wise loops (compiler generated).

6. Testing

  • Construction and splat/new, extract.
  • Arithmetic comparing lanes with scalar operations.
  • Load/Store.
  • Feature detection (if on x86, "sse" true).
  • Fallback: correct on non‑SIMD platforms.
