Blaze Phase 2b – Generics, Traits, and Impls

Previous: Phase 2a | Next: Phase 2c – Effects

Goal

Extend the compiler to support generic functions, generic structs/enums, trait declarations, trait implementations, trait bounds, dyn Trait objects, where clauses, and the coherence rules.

1. Generic Parameters

1.1 Syntax

generic_params = "[" generic_param { "," generic_param } [ "," ] "]"
generic_param  = ident [ ":" bound ] | "effect" ident
bound          = path { "+" path }

1.2 Semantics

  • Type parameters may have trait bounds.
  • Inference from argument types; explicit turbofish func::<T>(x).
  • Ambiguous types require explicit argument.

1.3 Monomorphisation

Separate code copy for each concrete type argument set. No runtime overhead.

2. Generic Structs and Enums

struct Vec<T> { data: *mut T, len: usize, cap: usize }
enum Option<T> { Some(T), None }

Type parameters in scope for fields and variants.

3. Traits

3.1 Trait Declaration

trait_decl = "trait" ident [ generic_params ] [ ":" bound ] "{" { trait_item } "}"
trait_item = fn_decl_no_body | const_decl | type_decl

3.2 @trivial Trait Attribute

@trivial(implementors) – closed‑world optimisation. Replaces dyn Trait with a match over an enum of listed implementors. No vtable; tag stored instead of vtable pointer. If implementors omitted, compiler collects all implementors in current crate.

3.3 Trait Bounds

T: Trait or where T: Trait.

3.4 Impl Blocks

impl_decl = "impl" [ generic_params ] ( type "{" { impl_item } "}" | bound "for" type "{" { impl_item } "}" )

3.5 Method Resolution

  1. Inherent methods.
  2. Trait methods.
  3. Fully‑qualified syntax for ambiguity.

3.6 dyn Trait

Dynamic dispatch via vtable (or tag+data for @trivial). Object safety rules enforced.

4. Coherence and Orphan Rules

  • Coherence: at most one impl per (Trait, Type).
  • Orphan: impl allowed only if Trait or Type defined in current crate.

5. Code Generation

  • Monomorphisation with name mangling.
  • Vtables for dyn Trait; no vtable for @trivial.
  • Indirect call for dyn; direct jump for @trivial.

6. Testing

  • Generic functions: instantiation, inference, ambiguous error.
  • Generic structs/enums: construction, field access, method calls.
  • Trait declarations/impls: inherent, trait, default methods.
  • Trait bounds: inline and where.
  • dyn Trait: coercion, dispatch, object safety.
  • Coherence/Orphan: conflicting impls, orphan violations.
  • @trivial: closed‑world dispatch, no vtable, compile‑time error for missing implementors.
  • Complexity: monomorphisation O(instantiations × size).
