Blaze Phase 3c – Extended Library: Dynamic Dispatch Helpers

Previous: Time | Next: Fixed‑Point

Goal

Utilities for dyn Trait: downcasting, type checking, vtable introspection.

1. DynTrait Type

pub struct DynTrait<T: ?Sized> { data: *mut u8, vtable: *const VTable }

Internal representation; compiler constructs automatically.

Raw Creation (unsafe)

pub unsafe fn from_raw_parts(data: *mut u8, vtable: *const VTable) -> Self;
pub fn data_ptr(&self) -> *mut u8;
pub fn vtable_ptr(&self) -> *const VTable;

2. VTable

pub struct VTable {
    drop: unsafe fn(*mut u8),
    size: usize,
    align: usize,
    methods: *const MethodTable,
}

MethodTable is opaque, compiler‑generated.

3. Downcasting

3.1 Any Trait

pub trait Any: 'static {}

Blanket implementation for all 'static types.

3.2 downcast_ref and downcast_mut 

pub fn downcast_ref<T: Any>(obj: &dyn Any) -> Option<&T>;
pub fn downcast_mut<T: Any>(obj: &mut dyn Any) -> Option<&mut T>;

Uses TypeId stored in vtable.

3.3 TypeId

pub struct TypeId(u64);
impl TypeId {
    pub fn of<T: Any>() -> TypeId;
}

Generated from mangled name hash (FNV‑1a).

4. Implementation Notes

  • Vtable includes TypeId for types implementing Any.
  • downcast compares TypeId stored in vtable with requested.

5. Testing

  • Coerce concrete type to dyn Trait, call method.
  • Downcast dyn Any back to concrete.
  • Incorrect downcast returns None.
  • TypeId uniqueness.
