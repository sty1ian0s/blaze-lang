Status

Normative for implementations that choose to support these features. None are required for core conformance.

A.1 Persistent Regions

A region annotated with @persistent may be mapped to non‑volatile storage (NVDIMM, battery‑backed RAM, flash‑backed memory) when supported. 

@persistent region { let data = Owned::new(vec![1, 2, 3]); }

Allocations in persistent regions survive process restarts. Runtime provides Region::open_persistent(name) to re‑open. Pointers are relocatable (indirection tables). Linear types are not automatically disposed on exit; commit or rollback via normal region end or panic.

A.2 Distributed Execution

spawn_on(node, actor) schedules an actor on a remote node. Returns network‑transparent Capability.

Serialises actor (fields) via blaze‑serde, transmits to remote runtime, deserialises and spawns. Local capability proxies sends. Remote unreachable → faulted state; NetworkError on sends. Supervision: remote actor's supervisor is local caller; panic escalation follows local rules (3 panics in 10 sec).

Configuration in blaze.toml:

[distributed]
nodes = ["192.168.1.10:9090", "192.168.1.11:9090"]

A.3 LLVM Backend

Alternative code generation via LLVM: blaze build --backend=llvm --release. Translates Blaze UIR to LLVM IR, uses LLVM optimisation pipeline. Preserves semantics, deterministic with --reproducible (fixed seeds, CRlibm).

A.4 C/C++ Transpiler (Community)

c2blaze translates a restricted subset of C/C++ to Blaze using Clang AST and LLVM IR. Detects patterns: local malloc/free → region, const T* → &T, trivially‑copyable structs → @copy, fixed‑size arrays → [T; N], null‑terminated strings → &str, error conventions → Result<T,E>. Unsafe code remains in unsafe blocks.
