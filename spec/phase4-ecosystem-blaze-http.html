Blaze Phase 4 – Ecosystem: HTTP

Previous: GUI | Next: HTTP/2

Goal

HTTP client and actor‑based server built on Blaze's core I/O and async primitives.

1. Core Types

pub enum HttpError { Io(std::io::Error), Parse(ParseError), Protocol(Text), Timeout }
pub enum Method { Get, Post, Put, Delete, Head, Options, Patch, Connect, Trace }
pub struct StatusCode(u16);
pub struct Headers { /* small‑vector */ }
pub struct Uri { scheme: Text, host: Text, port: u16, path: Text, query: Option<Text> }
pub enum Body { Empty, Bytes(Vec<u8>), Text(Text), Stream(Box<dyn BodyStream>) }

2. Client

pub struct Client { /* connection pool */ }
impl Client { pub fn new() -> Self; pub fn with_config(config: ClientConfig) -> Self; pub async fn send(&self, request: Request) -> Result<Response, HttpError>; pub async fn get(&self, uri: &str) -> Result<Response, HttpError>; }

3. Server

pub struct HttpServer { /* listens */ }
impl HttpServer { pub fn bind(addr: &str) -> Result<Self, HttpError>; pub async fn serve(self, handler: impl HttpHandler + Send + 'static) -> Result<(), HttpError>; }
pub trait HttpHandler: Send + 'static { fn handle(&self, request: Request) -> Response; }
pub struct Router { /* routes */ }
impl Router { pub fn new() -> Self; pub fn route(mut self, method: Method, path: &str, handler: fn(Request) -> Response) -> Self; }

4. Serde Integration

#[cfg(feature = "serde")] impl Request { pub fn json<T: Deserialize<'static>>(&self) -> Result<T, Error>; }
#[cfg(feature = "serde")] impl Response { pub fn json<T: Serialize>(body: &T) -> Response; }

5. Testing

  • Client: mock server, GET/POST, redirects, timeouts.
  • Server: bind, handler, response.
  • Router: route matching.
  • Error handling.
  • Actor isolation.
