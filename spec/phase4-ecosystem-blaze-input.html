Blaze Phase 4 – Ecosystem: Input

Previous: Image | Next: JNI

Goal

Unified, cross‑platform input abstraction for keyboard, mouse, touch, pen, and gamepad. Collects raw events from OS and delivers them as messages to the actor system. Zero‑copy, deterministic, low‑latency.

1. Input Events

pub enum InputEvent {
    Key(KeyEvent),
    Mouse(MouseEvent),
    Wheel(ScrollEvent),
    Touch(TouchEvent),
    Pen(PenEvent),
    Gamepad(GamepadEvent),
    Window(WindowEvent),
}

KeyEvent

pub struct KeyEvent { pub key: Key, pub state: ButtonState, pub modifiers: Modifiers, pub repeat: bool, pub scancode: u32 }
pub enum Key { A, B, C, ..., Digit0, F1, Escape, Tab, Shift, Control, Alt, Meta, Enter, Space, Backspace, Insert, Delete, Home, End, PageUp, PageDown, Left, Right, Up, Down, Numpad0, ..., VolumeUp, Mute, ... }

MouseEvent

pub struct MouseEvent { pub kind: MouseKind, pub button: MouseButton, pub position: (f64, f64), pub modifiers: Modifiers }
pub enum MouseKind { Moved, Pressed, Released, Entered, Exited }
pub enum MouseButton { Left, Right, Middle, Back, Forward, Other(u8) }

ScrollEvent

pub struct ScrollEvent { pub delta: (f64, f64), pub position: (f64, f64), pub modifiers: Modifiers }

TouchEvent

pub struct TouchEvent { pub phase: TouchPhase, pub touches: Vec<Touch> }
pub enum TouchPhase { Started, Moved, Ended, Cancelled }
pub struct Touch { pub id: u64, pub position: (f64, f64), pub pressure: Option<f64> }

PenEvent, GamepadEvent, WindowEvent

Analogous.
