Blaze Phase 3b – Systems Library: File System

Previous: Endianness | Next: Futures

Goal

File system operations: read_to_text, read, write, create_dir, metadata.

1. Public Functions

1.1 read_to_text

pub fn read_to_text(path: &str) -> Result<Text, io::Error>;

Read entire file as UTF‑8.

1.2 read

pub fn read(path: &str) -> Result<Vec<u8>, io::Error>;

Read entire file as bytes.

1.3 write

pub fn write(path: &str, data: &[u8]) -> Result<(), io::Error>;

Write bytes to file (truncate/create).

1.4 create_dir

pub fn create_dir(path: &str) -> Result<(), io::Error>;

Create directory; fails if exists.

1.5 metadata

pub fn metadata(path: &str) -> Result<Metadata, io::Error>;

2. Metadata Struct

pub struct Metadata {
    pub size: u64,
    pub is_file: bool,
    pub is_dir: bool,
}

3. Implementation Notes

  • Uses std::io::File internally.
  • read_to_text uses read_to_text from Read trait.
  • write retries short writes.
  • create_dir uses OS mkdir (Unix) or CreateDirectoryW (Windows).
  • metadata uses OS stat or GetFileAttributesEx.

4. Testing

  • read_to_text: write temp file, read back.
  • read: verify bytes.
  • write: read back and compare.
  • create_dir: verify exists via metadata.
  • metadata: check size, type.
  • Error handling: missing file, permission denied.
