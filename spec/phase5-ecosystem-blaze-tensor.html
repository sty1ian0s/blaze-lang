Blaze Phase 5 – Ecosystem: Tensor

Previous: ML | Next: Plot

Goal

Generic n‑dimensional array library with SoA‑aware storage, device abstraction (CPU/GPU), broadcasting, slicing, linear algebra, and automatic parallelisation.

1. Core Types

pub struct Tensor<T, const N: usize> {
    data: Owned<[T]>,
    shape: [usize; N],
    strides: [usize; N],
    device: Device,
}
pub struct TensorView<'a, T, const N: usize> {
    data: &'a [T],
    shape: [usize; N],
    strides: [usize; N],
}

2. Construction

impl<T: Numeric, const N: usize> Tensor<T, N> {
    pub fn new(shape: [usize; N], device: Device) -> Self;
    pub fn from_slice(slice: &[T], shape: [usize; N], device: Device) -> Self;
    pub fn zeros(shape: [usize; N], device: Device) -> Self;
    pub fn ones(shape: [usize; N], device: Device) -> Self;
    pub fn full(shape: [usize; N], value: T, device: Device) -> Self;
    pub fn eye(dim: usize, device: Device) -> Tensor<T, 2>;
    pub fn random(shape: [usize; N], device: Device) -> Self;
    pub fn from_fn(shape: [usize; N], f: impl Fn([usize; N]) -> T, device: Device) -> Self;
    pub fn to_device(&self, device: Device) -> Self;
}

3. Operations

  • Element‑wise: add, sub, mul, div, rem, neg, abs, sqrt, exp, log, pow, sin, cos, tan, relu, sigmoid, tanh, softmax.
  • Linear algebra: matmul, dot, transpose, permute, svd, inverse, solve.
  • Reductions: sum, mean, max, min, argmax.

4. Broadcasting

pub fn broadcast<const M: usize>(&self, target_shape: [usize; M]) -> TensorView<T, M>;

Lazy broadcasting via strides.

5. SoA for Structured Elements

StructuredTensor<S, const N: usize> for structs with @layout(soa).

6. Serialization

pub fn save(path: &str, tensor: &Tensor<T, N>) -> Result<(), TensorError>;
pub fn load(path: &str) -> Result<Tensor<T, N>, TensorError>;

7. Error Handling

pub enum TensorError {
    ShapeMismatch(Text), OutOfBounds, DeviceMismatch,
    NotSupported(Text), LayoutError(Text), Io(std::io::Error),
}
