Appendix E – Project Security Configuration

Previous: Appendix D | Next: Appendix F

Status

Normative. Defines the blaze.toml configuration schema for project‑wide security policies, contract/assertion handling in build profiles.

E.1 The blaze.toml Manifest

Every Blaze crate may contain a blaze.toml file in its root directory. Unknown keys cause a warning but not an error.

E.2 [security] Section

E.2.1 public_functions_must_have_contracts

  • Type: bool
  • Default: false
  • Description: If true, every public function must carry at least one requires or ensures clause, or be annotated with @no_contract (with a comment).

E.2.2 unsafe_requires_safety_comment

  • Type: bool
  • Default: false
  • Description: If true, every unsafe block must be immediately preceded by // SAFETY: comment. Rejects otherwise.

E.2.3 panic_policy

  • Type: string, either "unwind" or "abort"
  • Default: "unwind"
  • Description: "unwind" unwinds stack, calling Dispose; "abort" terminates immediately.

E.2.4 strict_narrowing

  • Type: bool
  • Default: false
  • Description: If true, narrowing as conversions are compile‑time errors unless inside unsafe or using try_into.

E.2.5 minimum_fuzz_iterations

  • Type: integer
  • Default: 0
  • Description: Minimum iterations for @fuzz harnesses during blaze test.

E.3 [profile] Section

E.3.1 keep_contracts

  • Type: bool
  • Default: true for debug, false for release
  • Description: When true, contract checks compiled into binary.

E.3.2 keep_assertions

  • Type: bool
  • Default: true for debug, false for release
  • Description: When true, debug_assert! compiled into binary.

E.3.3 Example

[profile.debug]
keep_contracts = true
keep_assertions = true

[profile.release]
keep_contracts = false
keep_assertions = false

E.4 Interaction with blaze fix

blaze fix --aot-ify may insert @no_contract and // SAFETY: comments automatically based on policies.

E.5 Testing

  • Public function without contract → reject.
  • unsafe without comment → reject.
  • panic_policy = "abort" → immediate termination.
  • strict_narrowing = true → narrowing cast error.
  • minimum_fuzz_iterations = 10 with 5 iterations → blaze test fails.
  • --release – contracts and debug_assert! absent from binary.
